<h2>PEP 429 – Python 3.4 릴리스 스케줄</h2>
<p>이 문서는 Python 3.4의 개발 및 릴리스 스케줄을 설명하며, 주로 PEP (Python Enhancement Proposal) 규모의 항목들에 초점을 맞춥니다.</p>

<h3>Python 3.4의 주요 기능</h3>
<p>Python 3.4 릴리스에 포함되었거나 최종적으로 구현된 PEP들은 다음과 같습니다.</p>
<p><strong>구현 완료/최종 PEPs:</strong></p>
<ul>
<li><strong>PEP 428:</strong> 객체 지향 파일 시스템 경로를 제공하는 "pathlib" 모듈 추가.</li>
<li><strong>PEP 435:</strong> 표준화된 "enum" 모듈 추가.</li>
<li><strong>PEP 436:</strong> 내장(builtins) 함수에 대한 introspection (내부 탐색) 정보 생성을 돕는 빌드 개선.</li>
<li><strong>PEP 442:</strong> 객체 종료(object finalization)에 대한 개선된 시맨틱스 (의미론).</li>
<li><strong>PEP 443:</strong> 표준 라이브러리에 단일 디스패치 제네릭 함수(single-dispatch generic functions) 추가.</li>
<li><strong>PEP 445:</strong> 사용자 정의 메모리 할당자를 구현하기 위한 새로운 C API.</li>
<li><strong>PEP 446:</strong> 서브프로세스(subprocesses)에서 파일 디스크립터(file descriptors)가 기본적으로 상속되지 않도록 변경.</li>
<li><strong>PEP 450:</strong> 새로운 "statistics" 모듈 추가.</li>
<li><strong>PEP 451:</strong> Python의 모듈 임포트 시스템을 위한 모듈 메타데이터 표준화.</li>
<li><strong>PEP 453:</strong> <code>pip</code> 패키지 관리자를 위한 번들 인스톨러 (bundled installer) 제공.</li>
<li><strong>PEP 454:</strong> Python 메모리 할당을 추적하기 위한 새로운 "tracemalloc" 모듈.</li>
<li><strong>PEP 456:</strong> Python 문자열 및 바이너리 데이터에 대한 새로운 해시 알고리즘.</li>
<li><strong>PEP 3154:</strong> <code>pickle</code> 된 객체를 위한 새롭고 개선된 프로토콜.</li>
<li><strong>PEP 3156:</strong> 비동기 I/O (asynchronous I/O)를 위한 새로운 프레임워크인 "asyncio" 모듈.</li>
</ul>
<p><strong>3.4 이후로 연기된 PEPs:</strong></p>
<ul>
<li><strong>PEP 431:</strong> 시간대 데이터베이스에 대한 개선된 지원.</li>
<li><strong>PEP 441:</strong> Python zip 애플리케이션 지원 개선.</li>
<li><strong>PEP 447:</strong> <code>__locallookup__</code> 메타클래스(metaclass) 메서드 지원.</li>
<li><strong>PEP 448:</strong> 추가적인 언패킹 일반화 (additional unpacking generalizations).</li>
<li><strong>PEP 455:</strong> 키 변환 딕셔너리 (key transforming dictionary).</li>
</ul>
