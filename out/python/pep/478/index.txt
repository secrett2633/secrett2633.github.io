<h2>PEP 478 – Python 3.5 릴리스 일정</h2>
<p>이 문서는 Python 3.5의 개발 및 릴리스 일정에 대해 설명하며, 주로 PEP(Python Enhancement Proposal)에 해당하는 주요 항목들을 다룹니다.</p>
<h3>초록 (Abstract)</h3>
<p>이 문서는 Python 3.5의 개발 및 릴리스 일정을 기술합니다. 일정은 주로 PEP 규모의 항목들에 초점을 맞춥니다.</p>
<h3>릴리스 관리자 및 팀 (Release Manager and Crew)</h3>
<ul>
<li><strong>3.5 릴리스 관리자:</strong> Larry Hastings</li>
<li><strong>Windows 설치 프로그램:</strong> Steve Dower</li>
<li><strong>Mac 설치 프로그램:</strong> Ned Deily</li>
<li><strong>문서화:</strong> Georg Brandl</li>
</ul>
<h3>릴리스 일정 (Release Schedule)</h3>
<p>Python 3.5는 현재 서비스 종료(end-of-life)에 도달했으며 은퇴(retired)되었습니다. 더 이상 릴리스는 없을 것입니다.</p>

<h3>3.5의 주요 기능 (Features for 3.5)</h3>
<p>Python 3.5에는 다음과 같은 주요 개선 및 기능들이 포함되었습니다.</p>
<ul>
<li><strong>PEP 441, 개선된 Python zip 애플리케이션 지원:</strong> Python이 zip 파일 내의 애플리케이션을 처리하는 방식이 개선되었습니다.</li>
<li><strong>PEP 448, 추가적인 언패킹 일반화:</strong> <code>*</code> 및 <code>**</code> 언패킹 연산자의 사용 범위가 확장되어 다양한 컨텍스트에서 더 유연하게 활용할 수 있게 되었습니다.</li>
<li><strong>PEP 461, bytes 및 bytearray 객체를 위한 "%-formatting":</strong> <code>bytes</code> 및 <code>bytearray</code> 객체에 대해 C 스타일의 <code>%</code> 포맷팅을 사용할 수 있게 되었습니다.</li>
<li><strong>PEP 465, 행렬 곱셈을 위한 새로운 연산자 ("@"):</strong> NumPy와 같은 라이브러리에서 행렬 곱셈을 명시적으로 수행하는 <code>@</code> 연산자가 도입되었습니다.</li>
<li><strong>PEP 471, os.scandir(), 빠른 새 디렉터리 순회 함수:</strong> 기존의 <code>os.listdir()</code>보다 더 효율적으로 디렉터리 항목을 순회할 수 있는 <code>os.scandir()</code> 함수가 추가되었습니다.</li>
<li><strong>PEP 475, 중단된 시스템 호출의 자동 재시도 지원 추가:</strong> 시스템 호출이 인터럽트(interrupted)되었을 때 자동으로 재시도하는 메커니즘이 추가되어 견고성이 향상되었습니다.</li>
<li><strong>PEP 479, 제너레이터 내 StopIteration 처리 변경:</strong> 제너레이터(Generator) 내부에서 <code>StopIteration</code> 예외가 발생했을 때의 동작 방식이 변경되어, 의도치 않은 버그를 방지할 수 있게 되었습니다.</li>
<li><strong>PEP 484, typing 모듈, 타입 어노테이션을 위한 새로운 표준:</strong> 정적 타입 힌트(Type Hint)를 위한 <code>typing</code> 모듈과 타입 어노테이션(Type Annotation) 표준이 도입되었습니다.</li>
<li><strong>PEP 485, math.isclose(), 근사치 동등성 테스트 함수:</strong> 두 부동 소수점 숫자가 거의 같은지(approximate equality)를 테스트하는 <code>math.isclose()</code> 함수가 추가되었습니다.</li>
<li><strong>PEP 486, 가상 환경을 인식하는 Windows Python 런처:</strong> Windows의 Python 런처(<code>py.exe</code>)가 가상 환경(virtual environment)을 더 잘 인식하고 활성화할 수 있도록 개선되었습니다.</li>
<li><strong>PEP 488, .pyo 파일 제거:</strong> 최적화된 바이트코드 파일을 나타내는 <code>.pyo</code> 파일이 제거되고, <code>.pyc</code> 파일이 최적화된 바이트코드도 포함하도록 변경되었습니다.</li>
<li><strong>PEP 489, 확장 모듈 로딩을 위한 새롭고 개선된 메커니즘:</strong> C 확장 모듈(extension module)을 로드하는 메커니즘이 개선되어 더 효율적이고 유연해졌습니다.</li>
<li><strong>PEP 492, async 및 await 구문을 가진 코루틴:</strong> 비동기 프로그래밍(Asynchronous Programming)을 위한 <code>async</code> 및 <code>await</code> 키워드를 사용한 네이티브 코루틴(Coroutines) 문법이 도입되었습니다.</li>
</ul>
