<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0315/" target="_blank" rel="noopener noreferrer">PEP 315 - Enhanced While Loop</a></p>
<p><strong>상태:</strong> Rejected | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 25-Apr-2003</p>
</blockquote>
<h3 id="개요-abstract"><a href="#개요-abstract">개요 (Abstract)</a></h3>
<p>이 PEP는 <code>while</code> 루프의 시작 부분에 선택적인 "do" 절을 추가하여 루프 코드를 더 명확하게 만들고, 코드 중복으로 인한 오류를 줄이는 것을 제안합니다.</p>
<h3 id="공지-notice"><a href="#공지-notice">공지 (Notice)</a></h3>
<p>이 PEP는 거부되었습니다. 2006년부터 이 PEP는 보류 상태였습니다. 2009년 4월에 PEP를 부활시키려는 후속 노력은 다음 형태와 경쟁할 만한 문법이 나오지 않아 성공하지 못했습니다:</p>
<pre><code class="language-python">while True:
    &#x3C;setup code>
    if not &#x3C;condition>:
        break
    &#x3C;loop body>
</code></pre>
<p>PEP에서 제안된 것과는 다른 기본 <code>do-while</code> 루프를 위한 대체 문법이 발견되었으나, 조건이 맨 위에 있었기 때문에 거의 지지를 받지 못했습니다:</p>
<pre><code class="language-python">do ... while &#x3C;cond>:
    &#x3C;loop body>
</code></pre>
<p>따라서, <code>do-while</code> 루프가 적절했을 상황에서는 언어 사용자들에게 <code>while True</code> 형태에 <code>if-break</code>를 내부에 사용하는 것을 권장하고 있습니다.</p>
<h3 id="동기-motivation"><a href="#동기-motivation">동기 (Motivation)</a></h3>
<p><code>while</code> 루프 조건이 평가되기 전에 특정 코드를 실행해야 하는 경우가 자주 있습니다. 이러한 코드는 종종 루프 바깥에 한 번 실행되는 설정 코드로서 중복되어 나타납니다:</p>
<pre><code class="language-python">&#x3C;setup code>
while &#x3C;condition>:
    &#x3C;loop body>
    &#x3C;setup code>
</code></pre>
<p>문제는 코드가 중복되면 한 인스턴스가 변경될 때 다른 인스턴스가 변경되지 않아 오류의 원인이 될 수 있다는 것입니다. 또한, 두 번째 설정 코드 인스턴스는 루프 끝에 오기 때문에 그 목적이 명확하지 않습니다.</p>
<p>코드 중복을 방지하기 위해 루프 조건을 헬퍼 함수나 루프 본문의 <code>if</code> 문으로 이동하는 것도 가능합니다. 그러나 <code>while</code> 키워드로부터 루프 조건을 분리하면 루프의 동작이 덜 명확해집니다:</p>
<pre><code class="language-python">def helper(args):
    &#x3C;setup code>
    return &#x3C;condition>

while helper(args):
    &#x3C;loop body>
</code></pre>
<p>이 마지막 형태는 루프의 <code>else</code> 절을 <code>if</code> 문의 본문에 추가해야 하는 추가적인 단점이 있어, 루프의 동작을 더욱 모호하게 만듭니다:</p>
<pre><code class="language-python">while True:
    &#x3C;setup code>
    if not &#x3C;condition>:
        break
    &#x3C;loop body>
</code></pre>
<p>이 PEP는 이러한 문제를 해결하기 위해 <code>while</code> 루프에 선택적인 절을 추가하여 설정 코드를 자연스러운 방식으로 표현할 수 있도록 제안했습니다:</p>
<pre><code class="language-python">do:
    &#x3C;setup code>
while &#x3C;condition>:
    &#x3C;loop body>
</code></pre>
<p>이 방식은 루프 조건을 마땅히 있어야 할 <code>while</code> 키워드와 함께 유지하며 코드 중복을 요구하지 않습니다.</p>
<h3 id="문법-syntax"><a href="#문법-syntax">문법 (Syntax)</a></h3>
<p><code>while</code> 문의 기존 문법은 다음과 같습니다:</p>
<pre><code>while_stmt : "while" expression ":" suite ["else" ":" suite]
</code></pre>
<p>이 PEP는 이를 다음과 같이 확장하는 것을 제안했습니다:</p>
<pre><code>while_stmt : ["do" ":" suite] "while" expression ":" suite ["else" ":" suite]
</code></pre>
<h3 id="break와-continue의-의미-semantics-of-break-and-continue"><a href="#break와-continue의-의미-semantics-of-break-and-continue"><code>break</code>와 <code>continue</code>의 의미 (Semantics of break and continue)</a></h3>
<p>제안된 <code>do-while</code> 루프에서 <code>break</code> 문은 표준 <code>while</code> 루프와 동일하게 동작합니다. 즉, 루프 조건을 평가하거나 <code>else</code> 절을 실행하지 않고 즉시 루프를 종료합니다.</p>
<p><code>do-while</code> 루프 내의 <code>continue</code> 문은 <code>while</code> 조건 검사 지점으로 점프합니다.</p>
<p>일반적으로 <code>while</code> 스위트(suite)가 비어 있는 경우(<code>pass</code> 문) <code>do-while</code> 루프와 <code>break</code>, <code>continue</code> 문은 다른 언어의 <code>do-while</code> 의미와 일치해야 합니다.</p>
<p>마찬가지로 <code>do</code> 스위트가 비어 있는 경우 <code>do-while</code> 루프와 <code>break</code>, <code>continue</code> 문은 일반 <code>while</code> 루프에서 발견되는 동작과 일치해야 합니다.</p>
<h3 id="future-statement"><a href="#future-statement">Future Statement</a></h3>
<p>새로운 키워드 "do" 때문에 <code>do-while</code> 형태를 사용하려면 초기에는 <code>from __future__ import do_while</code> 문이 필요할 것이었습니다.</p>
<h3 id="구현-implementation"><a href="#구현-implementation">구현 (Implementation)</a></h3>
<p>이 PEP의 첫 번째 구현은 <code>do-while</code> 루프를 루프를 종료하는 테스트가 있는 무한 루프로 컴파일할 수 있습니다.</p>
