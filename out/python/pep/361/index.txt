<h1>PEP 361 – Python 2.6 및 3.0 릴리스 스케줄</h1>
<p><em>저자: Neal Norwitz, Barry Warsaw</em>
<em>상태: 최종 (Final)</em>
<em>유형: 정보성 (Informational)</em>
<em>주제: 릴리스 (Release)</em>
<em>생성일: 2006년 6월 29일</em>
<em>Python 버전: 2.6, 3.0</em>
<em>최종 변경일: 2008년 3월 17일</em></p>
<h2>초록 (Abstract)</h2>
<p>이 문서는 Python 2.6 및 Python 3.0의 개발 및 릴리스 스케줄을 설명합니다. 스케줄은 주로 PEP 규모의 항목들에 초점을 맞춥니다. 작은 기능들은 첫 번째 베타 릴리스까지 추가될 수 있으며, 버그는 최종 릴리스까지 수정될 수 있습니다.</p>
<p>최소 두 번의 알파 릴리스, 두 번의 베타 릴리스, 그리고 한 번의 릴리스 후보(Release Candidate, RC)가 계획되었습니다. 릴리스는 2008년 10월로 예정되었습니다.</p>
<p>Python 2.6은 Python 2 시리즈의 다음 발전일 뿐만 아니라, 개발자들이 Python 3.0을 위해 코드를 준비하는 데 도움을 주는 "과도기적 릴리스(transitional release)"입니다. 이러한 이유로 Python 3.0의 많은 기능이 2.6으로 백포팅(backported)되었습니다. 따라서 두 버전을 동시에 릴리스하는 것이 합리적입니다. 이는 Python 1.6과 2.0 릴리스에서 선례가 있었습니다.</p>
<p>RC 릴리스 전까지, Python 2.6과 3.0은 월별 릴리스 주기로 "록스텝(lockstep)" 방식으로 함께 릴리스되었습니다. 릴리스는 베타 테스트 주기 동안 매월 첫째 주 수요일에 진행되었습니다. Python 2.6이 더 빨리 준비되었고, 외부 마감 기한을 맞추기 위해 RC 릴리스를 분리하기로 결정했습니다. 이에 따라 Python 2.6 최종 버전은 Python 3.0 최종 버전보다 2주 먼저 출시되도록 계획되었습니다.</p>

<li><strong>Windows 인스톨러:</strong> Martin v. Loewis</li>

</ul>
<h2>릴리스 수명 (Release Lifespan)</h2>
<ul>
<li><strong>Python 3.0:</strong> 더 이상 어떤 목적으로도 유지보수되지 않습니다.</li>
<li><strong>Python 2.6:</strong> Python 2.6.9는 Python 2.6 시리즈의 최종 보안 전용 소스 전용 유지보수 릴리스입니다. 2013년 10월 29일 릴리스와 함께 Python 2.6에 대한 모든 공식 지원이 종료되었습니다. Python 2.6은 더 이상 어떤 목적으로도 유지보수되지 않습니다.</li>
</ul>

<ul>
<li>2008년 2월 29일: Python 2.6a1 및 3.0a3 릴리스</li>
<li>2008년 4월 2일: Python 2.6a2 및 3.0a4 릴리스</li>
<li>2008년 5월 8일: Python 2.6a3 및 3.0a5 릴리스</li>
<li>2008년 6월 18일: Python 2.6b1 및 3.0b1 릴리스</li>
<li>2008년 7월 17일: Python 2.6b2 및 3.0b2 릴리스</li>
<li>2008년 8월 20일: Python 2.6b3 및 3.0b3 릴리스</li>
<li>2008년 9월 12일: Python 2.6rc1 릴리스</li>
<li>2008년 9월 17일: Python 2.6rc2 및 3.0rc1 릴리스</li>
<li>2008년 10월 1일: Python 2.6 최종 버전 릴리스</li>
<li>2008년 11월 6일: Python 3.0rc2 릴리스</li>

<li>2013년 10월 29일: Python 2.6.9 최종 버전 릴리스 (보안 전용)</li>
</ul>
<h2>Python 3.0의 완료된 기능 (Completed features for 3.0)</h2>
<p>Python 3.0 프로젝트에 대한 자세한 내용은 <a href="https://peps.python.org/pep-3000/">PEP 3000</a> 및 <a href="https://peps.python.org/pep-3100/">PEP 3100</a>을 참조하십시오.</p>
<h2>Python 2.6의 완료된 기능 (Completed features for 2.6)</h2>
<p>Python 2.6에 포함된 주요 기능 및 변경 사항은 다음과 같습니다:</p>
<h3>PEP 목록</h3>
<ul>
<li><strong>PEP 352:</strong> 문자열 예외(string exception)를 발생시키면 <code>TypeError</code>가 발생합니다. 문자열 예외를 catch하려고 하면 <code>DeprecationWarning</code>이 발생합니다. <code>BaseException.message</code>가 Deprecated되었습니다.</li>
<li><strong>PEP 358:</strong> "bytes" 객체 도입.</li>
<li><strong>PEP 366:</strong> 메인 모듈의 명시적 상대 임포트(explicit relative imports).</li>
<li><strong>PEP 370:</strong> 사용자별 <code>site-packages</code> 디렉터리.</li>
<li><strong>PEP 3112:</strong> Python 3000의 바이트 리터럴(Bytes literals).</li>
<li><strong>PEP 3127:</strong> 정수 리터럴 지원 및 문법.</li>
<li><strong>PEP 371:</strong> <code>multiprocessing</code> 패키지 추가.</li>
</ul>
<h3>표준 라이브러리의 새로운 모듈</h3>
<ul>
<li><code>json</code></li>
<li>향상된 <code>turtle</code> 모듈</li>
<li><code>ast</code></li>
</ul>
<h3>표준 라이브러리에서 Deprecated된 모듈 및 함수</h3>
<ul>
<li><code>buildtools</code>, <code>cfmfile</code>, <code>commands.getstatus()</code>, <code>macostools.touched()</code>, <code>md5</code>, <code>MimeWriter</code>, <code>mimify</code>, <code>popen2</code>, <code>os.popen[234]()</code>, <code>posixfile</code>, <code>sets</code>, <code>sha</code></li>
</ul>
<h3>표준 라이브러리에서 제거된 모듈</h3>
<ul>
<li><code>gopherlib</code>, <code>rgbimg</code>, <code>macfs</code></li>
</ul>
<h3>Python 3000에서 제거된 기능에 대한 경고 (Warnings for features removed in Py3k)</h3>
<ul>
<li><strong>내장 함수:</strong> <code>apply</code>, <code>callable</code>, <code>coerce</code>, <code>dict.has_key</code>, <code>execfile</code>, <code>reduce</code>, <code>reload</code></li>
<li>백틱(<code>`</code>) 및 <code>&#x3C;></code> 연산자</li>
<li><code>xrange</code>에 대한 float 인자</li>
<li><code>coerce</code> 및 관련 함수들</li>
<li>기본 비교(<code>__cmp__</code>가 없는 경우)</li>
<li><code>{}.has_key()</code></li>
<li><code>file.xreadlines</code></li>
<li><code>print()</code> 함수의 <code>softspace</code> 제거</li>
<li>PEP 4/PEP 3100/PEP 3108로 인한 모듈 제거</li>
</ul>
<h3>기타 주요 기능</h3>
<ul>
<li><code>with / as</code>가 키워드가 되었습니다.</li>
<li><code>dir()</code> 함수를 제어하기 위한 <code>__dir__()</code> 특별 메서드가 추가되었습니다.</li>
<li>AtheOS 지원이 중단되었습니다.</li>
<li><code>warnings</code> 모듈이 C로 구현되었습니다.</li>
<li><code>compile()</code>이 AST(Abstract Syntax Tree)를 받아 바이트 코드로 변환할 수 있습니다.</li>
</ul>
<h2>Python 2.6의 가능한 기능 (Possible features for 2.6)</h2>
<p>새로운 기능은 <code>alpha2</code> 릴리스 이전에 구현되어야 하며, 특히 C 수정 사항이나 동작 변경 사항이 그렇습니다. 새로운 기능은 <code>beta1</code> 릴리스 이전에 구현되어야 하며, 그렇지 않으면 릴리스 관리자의 승인이 필요합니다.</p>
<p>이 문서는 릴리스 당시 2.6에 포함될 예정이었던 몇 가지 PEP와 논의 중인 기타 기능 및 변경 사항을 나열했습니다.</p>
<ul>
<li><code>distutils</code> 대체 (PEP 필요)</li>
<li>새로운 표준 라이브러리 모듈: <code>winerror</code>, <code>setuptools</code></li>
<li><code>DeprecationWarning</code>을 발생시킬 모듈: <code>rfc822</code>, <code>mimetools</code>, <code>multifile</code>, <code>compiler</code> 패키지</li>
<li>Py3k에서 제거될 기능에 대한 경고 추가: <code>__getslice__</code>, <code>__setslice__</code>, <code>__delslice__</code>, <code>PyArgs_ParseTuple</code>의 float 인자, <code>__cmp__</code> 등 비교 변경 사항, 정수 나눗셈, <code>PendingDeprecationWarnings</code>, <code>zip()</code> 결과 사용 방식 변경, <code>exec</code> 문, <code>func_*</code>로 시작하는 함수 속성, <code>L</code> 접미사 long 리터럴, <code>__nonzero__</code>를 <code>__bool__</code>로 이름 변경 등.</li>
</ul>
<h2>2.7로 연기된 기능 (Deferred until 2.7)</h2>
<p>없음.</p>
