<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0686/" target="_blank" rel="noopener noreferrer">PEP 686 - Make UTF-8 mode default</a></p>
<p><strong>상태:</strong> Accepted | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 18-Mar-2022</p>
</blockquote>
<h2 id="pep-686-utf-8-모드-기본값으로-설정"><a href="#pep-686-utf-8-모드-기본값으로-설정">PEP 686 – UTF-8 모드 기본값으로 설정</a></h2>
<ul>
<li><strong>작성자</strong> : Inada Naoki</li>
<li><strong>상태</strong> : Accepted (승인됨)</li>
<li><strong>유형</strong> : Standards Track (표준 트랙)</li>
<li><strong>생성일</strong> : 2022년 3월 18일</li>
<li><strong>Python 버전</strong> : 3.15</li>
<li><strong>해결</strong> : Discourse 메시지</li>
</ul>
<h3 id="개요-abstract"><a href="#개요-abstract">개요 (Abstract)</a></h3>
<p>이 PEP(Python Enhancement Proposal)는 Python의 UTF-8 모드를 기본값으로 활성화할 것을 제안합니다. 이 변경으로 인해, 파이썬은 파일, 표준 입출력(stdio), 그리고 파이프(pipe)의 기본 인코딩으로 일관되게 UTF-8을 사용하게 됩니다.</p>
<h3 id="동기-motivation"><a href="#동기-motivation">동기 (Motivation)</a></h3>
<p>UTF-8은 사실상 텍스트 인코딩의 표준이 되었습니다.</p>
<ul>
<li>Python 소스 파일의 기본 인코딩은 UTF-8입니다.</li>
<li>JSON, TOML, YAML과 같은 데이터 형식은 UTF-8을 사용합니다.</li>
<li>Visual Studio Code 및 Windows 메모장을 포함한 대부분의 텍스트 에디터가 기본적으로 UTF-8을 사용합니다.</li>
<li>대부분의 웹사이트와 인터넷상의 텍스트 데이터는 UTF-8을 사용합니다.</li>
<li>Node.js, Go, Rust, Java를 포함한 많은 인기 프로그래밍 언어도 기본적으로 UTF-8을 사용합니다.</li>
</ul>
<p>파이썬의 기본 인코딩을 UTF-8로 변경하면 이러한 다른 시스템 및 언어와의 상호 운용성이 더욱 쉬워집니다.</p>
<p>또한, 유닉스(Unix)를 사용하는 많은 파이썬 개발자들이 기본 인코딩이 플랫폼에 따라 다르다는 사실을 잊곤 합니다. 그들은 UTF-8로 인코딩된 텍스트 파일(예: JSON, TOML, Markdown, Python 소스 파일)을 읽을 때 <code>encoding="utf-8"</code> 옵션을 생략하는 경우가 많습니다. 이러한 일관성 없는 기본 인코딩은 많은 버그를 유발합니다.</p>
<h3 id="명세-specification"><a href="#명세-specification">명세 (Specification)</a></h3>
<h4 id="utf-8-모드-기본값으로-활성화-enable-utf-8-mode-by-default"><a href="#utf-8-모드-기본값으로-활성화-enable-utf-8-mode-by-default">UTF-8 모드 기본값으로 활성화 (Enable UTF-8 mode by default)</a></h4>
<p>Python 3.15부터 파이썬은 기본적으로 UTF-8 모드를 활성화합니다.</p>
<p>사용자는 <code>PYTHONUTF8=0</code> 환경 변수를 설정하거나 <code>-X utf8=0</code> 옵션을 사용하여 여전히 UTF-8 모드를 비활성화할 수 있습니다.</p>
<h4 id="localegetencoding"><a href="#localegetencoding"><code>locale.getencoding()</code></a></h4>
<p>UTF-8 모드는 <code>locale.getpreferredencoding(False)</code>에 영향을 미치므로, UTF-8 모드와 관계없이 로케일(locale) 인코딩을 가져올 수 있는 API가 필요합니다.</p>
<p>이를 위해 <code>locale.getencoding()</code> API가 추가됩니다. 이 함수도 로케일 인코딩을 반환하지만, UTF-8 모드의 영향을 무시합니다.</p>
<p><code>warn_default_encoding</code> 옵션이 지정되면, <code>locale.getpreferredencoding()</code>은 <code>open()</code> 함수와 유사하게 <code>EncodingWarning</code>을 발생시킬 것입니다 (PEP 597 참조). 이 API는 Python 3.11에 추가되었습니다.</p>
<h4 id="encodinglocale-옵션-수정-fixing-encodinglocale-option"><a href="#encodinglocale-옵션-수정-fixing-encodinglocale-option"><code>encoding="locale"</code> 옵션 수정 (Fixing encoding="locale" option)</a></h4>
<p>PEP 597은 <code>TextIOWrapper</code>에 <code>encoding="locale"</code> 옵션을 추가했습니다. 이 옵션은 로케일 인코딩을 명시적으로 지정하는 데 사용됩니다. <code>TextIOWrapper</code>는 기본 텍스트 인코딩과 관계없이 이 옵션이 지정되면 로케일 인코딩을 사용해야 합니다.</p>
<p>하지만 현재는 <code>encoding="locale"</code>이 지정되더라도 UTF-8 모드에서는 <code>TextIOWrapper</code>가 "UTF-8"을 사용합니다. 이 동작은 PEP 597의 동기와 일치하지 않습니다. 이는 파이썬이 기본 텍스트 인코딩을 변경할 때 UTF-8 모드를 기본값으로 만들 것을 예상하지 못했기 때문입니다.</p>
<p>이러한 불일치는 UTF-8 모드를 기본값으로 만들기 전에 수정되어야 합니다. <code>TextIOWrapper</code>는 UTF-8 모드에서도 <code>encoding="locale"</code>이 전달될 때 로케일 인코딩을 사용해야 합니다. 이 문제는 Python 3.11에서 수정되었습니다.</p>
<h3 id="하위-호환성-backward-compatibility"><a href="#하위-호환성-backward-compatibility">하위 호환성 (Backward Compatibility)</a></h3>
<p>대부분의 유닉스(Unix) 시스템은 UTF-8 로케일을 사용하며, 로케일이 C 또는 POSIX일 때 파이썬은 UTF-8 모드를 활성화합니다. 따라서 이 변경 사항은 주로 윈도우(Windows) 사용자에게 영향을 미칩니다.</p>
<p>파이썬 프로그램이 기본 인코딩에 의존하는 경우, 이 변경으로 인해 <code>UnicodeError</code>, 깨진 문자(mojibake) 또는 심지어 조용한 데이터 손상이 발생할 수 있습니다. 따라서 이 변경 사항은 크게 공지되어야 합니다.</p>
<p>이러한 하위 호환성 문제를 해결하기 위한 지침은 다음과 같습니다:</p>
<ol>
<li><strong>UTF-8 모드 비활성화</strong> : <code>PYTHONUTF8=0</code> 또는 <code>-X utf8=0</code>을 사용하여 UTF-8 모드를 비활성화합니다.</li>
<li><strong><code>EncodingWarning</code> 사용</strong> : <code>EncodingWarning</code> (PEP 597)을 사용하여 UTF-8 모드가 영향을 미치는 모든 위치를 찾습니다.</li>
<li><strong>인코딩 옵션 명시</strong> :
<ul>
<li><code>encoding</code> 옵션이 생략된 경우, <code>encoding="utf-8"</code> 또는 <code>encoding="locale"</code> 사용을 고려합니다.</li>
<li><code>locale.getpreferredencoding()</code>이 사용된 경우, <code>"utf-8"</code> 또는 <code>locale.getencoding()</code> 사용을 고려합니다.</li>
</ul>
</li>
<li><strong>UTF-8 모드로 애플리케이션 테스트</strong> : 변경 사항을 적용한 후 UTF-8 모드를 활성화하여 애플리케이션을 테스트합니다.</li>
</ol>
<h4 id="선행-사례-preceding-examples"><a href="#선행-사례-preceding-examples">선행 사례 (Preceding examples)</a></h4>
<ul>
<li>루비(Ruby)는 2020년 Ruby 3.0에서 윈도우 환경의 기본 <code>external_encoding</code>을 UTF-8로 변경했습니다.</li>
<li>자바(Java)는 2022년 JDK 18에서 기본 텍스트 인코딩을 UTF-8로 변경했습니다.</li>
</ul>
<p>두 언어 모두 하위 호환성을 위한 옵션을 제공하지만, 파이썬의 PEP 597 <code>EncodingWarning</code>과 같은 경고는 제공하지 않습니다.</p>
<h3 id="기각된-대안-rejected-alternative"><a href="#기각된-대안-rejected-alternative">기각된 대안 (Rejected Alternative)</a></h3>
<h4 id="암시적-인코딩-사용-중단-deprecate-implicit-encoding"><a href="#암시적-인코딩-사용-중단-deprecate-implicit-encoding">암시적 인코딩 사용 중단 (Deprecate implicit encoding)</a></h4>
<p>기본 인코딩 사용을 더 이상 권장하지 않는 방안이 고려되었습니다.</p>
<p>그러나 기본 인코딩이 ASCII 텍스트를 읽고 쓰는 데만 사용되는 경우가 많습니다. 또한, 이러한 경고는 유닉스에서 실행되는 비-크로스 플랫폼(non-cross platform) 애플리케이션에는 유용하지 않습니다.</p>
<p>따라서 사용자에게 모든 곳에서 인코딩을 명시하도록 강제하는 것은 너무 번거롭습니다. 많은 <code>DeprecationWarning</code>을 발생시키면 사용자들이 경고를 무시하게 될 것입니다.</p>
<p>PEP 387은 하위 호환되지 않는 변경 사항에 대해 경고를 추가하도록 요구하지만, <code>DeprecationWarning</code> 사용을 필수로 하지는 않습니다. 따라서 선택적 <code>EncodingWarning</code>을 사용하는 것은 PEP 387을 위반하지 않습니다.</p>
<p>자바(Java)도 JEP 400에서 이 아이디어를 기각했습니다.</p>
<h4 id="파이프pipe에-pythonioencoding-사용-use-pythonioencoding-for-pipes"><a href="#파이프pipe에-pythonioencoding-사용-use-pythonioencoding-for-pipes">파이프(PIPE)에 <code>PYTHONIOENCODING</code> 사용 (Use PYTHONIOENCODING for PIPEs)</a></h4>
<p>하위 호환성 문제를 완화하기 위해 <code>subprocess</code> 모듈의 파이프 기본 인코딩으로 <code>PYTHONIOENCODING</code>을 사용하는 방안이 고려되었습니다.</p>
<p>이 아이디어를 사용하면 사용자는 UTF-8 모드에서도 <code>subprocess.Popen(text=True)</code>에 레거시(legacy) 인코딩을 사용할 수 있습니다.</p>
<p>하지만 이 아이디어는 "기본 인코딩"을 복잡하게 만들고, 이 또한 하위 호환되지 않는 변경입니다.</p>
<p>따라서 이 아이디어는 기각되었습니다. 사용자는 <code>text=True</code>를 <code>encoding="utf-8"</code> 또는 <code>encoding="locale"</code>로 대체할 때까지 UTF-8 모드를 비활성화할 수 있습니다.</p>
<h3 id="교육-방법-how-to-teach-this"><a href="#교육-방법-how-to-teach-this">교육 방법 (How to teach this)</a></h3>
<p><strong>새로운 사용자(New users)의 경우:</strong> 이 변경은 가르쳐야 할 내용을 줄여줍니다. 사용자들은 첫해에 텍스트 인코딩에 대해 배울 필요가 없으며, 비-UTF-8 텍스트 파일을 사용해야 할 때 배우면 됩니다.</p>
<p><strong>기존 사용자(Existing users)의 경우:</strong> "하위 호환성" 섹션을 참조하십시오.</p>
