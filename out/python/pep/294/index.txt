<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0294/">PEP 294 - Type Names in the types Module</a></p>
<p><strong>상태:</strong> Rejected | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 19-Jun-2002</p>
</blockquote>
<h1>PEP 294 – <code>types</code> 모듈의 타입 이름</h1>
<ul>
<li><strong>작성자:</strong> Oren Tirosh</li>
<li><strong>상태:</strong> Rejected (거부됨)</li>
<li><strong>유형:</strong> Standards Track</li>
<li><strong>생성일:</strong> 2002년 6월 19일</li>
<li><strong>Python 버전:</strong> 2.5</li>
</ul>
<h2>개요 (Abstract)</h2>
<p>이 PEP는 <code>types</code> 모듈 내의 모든 기본(basic) Python 타입에 대해 타입 이름과 일치하는 심볼을 <code>types</code> 모듈에 추가할 것을 제안합니다.</p>
<p>예를 들어:</p>
<ul>
<li><code>types.IntegerType</code> -> <code>types.int</code></li>
<li><code>types.FunctionType</code> -> <code>types.function</code></li>
<li><code>types.TracebackType</code> -> <code>types.traceback</code></li>
</ul>
<p>현재 <code>types</code> 모듈에 있는 길고 대문자로 시작하는 이름들은 deprecated(사용 중단)될 예정이었습니다. 이 변경을 통해 <code>types</code> 모듈은 <code>new</code> 모듈을 대체할 수 있으며, <code>new</code> 모듈은 deprecated되고 PEP 4에 등재될 예정이었습니다.</p>
<h2>결정 (Pronouncement)</h2>
<p>타입 이름들을 중앙 집중식으로 저장하는 방식은 실수였습니다. <code>types</code> 모듈이나 <code>new</code> 모듈 모두 Python 3.0으로 계승되어서는 안 됩니다.</p>
<p>그러므로, 제안된 업데이트를 이 모듈들에 적용하는 것은 타당하지 않습니다. 이는 아무런 이점 없이 혼란만 야기할 것입니다.</p>
<p>대신, 일부 내부 타입(프레임, 함수 등)이 해당 모듈 외에는 존재하지 않는다는 문제는 <code>__builtin__</code> 또는 <code>sys</code> 모듈에 추가함으로써 해결될 수 있습니다. 이는 Python 3.0으로의 더 순조로운 전환을 제공할 것입니다.</p>
<h2>배경 (Rationale)</h2>
<p>동일한 객체에 두 가지 이름 세트를 사용하는 것은 불필요하며 혼란을 야기합니다.</p>
<p>Python 2.2 이전 버전에서는 많은 타입 이름과 일치하는 심볼들이 해당 타입의 팩토리 함수(factory function)에 의해 사용되었습니다. 이제 모든 기본 타입은 해당 팩토리 함수와 통합되었으므로, 타입 이름은 타입 객체를 일관되게 참조하는 데 사용할 수 있게 되었습니다.</p>
<p>대부분의 타입은 내장(builtins)으로 또는 <code>new</code> 모듈에서 접근할 수 있지만, <code>traceback</code>이나 <code>generator</code>와 같은 일부 타입은 <code>types</code> 모듈을 통해서만 타입 이름과 일치하지 않는 이름으로 접근할 수 있습니다. 이 PEP는 모든 기본 타입을 단일 이름 세트로 접근할 수 있는 통일된 방법을 제공하고자 했습니다.</p>
<h2>명세 (Specification)</h2>
<p><code>types</code> 모듈은 다음 테스트를 통과해야 했습니다.</p>
<pre><code class="language-python">import types
for t in vars(types).values():
    if type(t) is type:
        assert getattr(types, t.__name__) is t
</code></pre>
<p><code>class</code>, <code>instance method</code>, <code>dict-proxy</code>와 같은 타입들은 이미 유효한 Python 식별자인 <code>classobj</code>, <code>instancemethod</code>, <code>dictproxy</code>로 이름이 변경되어 이 변경이 가능하게 되었습니다.</p>
<h2>하위 호환성 (Backward compatibility)</h2>
<p>기존의 긴 이름들이 광범위하게 사용되고 있기 때문에, 향후 버전에서 <code>types</code> 모듈에서 해당 이름들을 실제로 제거할 계획은 없었습니다. 그러나 새로운 코드에서 사용을 지양하도록 문서 및 라이브러리 소스에서 긴 이름들을 변경해야 했습니다.</p>
