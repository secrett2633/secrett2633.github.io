<h2 id="개요"><a href="#개요">개요</a></h2>
<p>이 문서는 Python 3.11의 개발 및 릴리스 일정을 기술합니다. 일정은 주로 PEP 규모의 항목들을 다룹니다.</p>

<h3 id="311-수명-주기-lifespan"><a href="#311-수명-주기-lifespan">3.11 수명 주기 (Lifespan)</a></h3>
<p>Python 3.11은 약 18개월 동안 약 2개월마다 버그픽스 업데이트를 받았습니다. 3.12.0 final 릴리스 이후 얼마 지나지 않아 9번째이자 마지막 3.11 버그픽스 업데이트가 릴리스되었습니다. 그 이후에는 3.11.0 final 릴리스 후 5년까지, 즉 약 2027년 10월까지 보안 업데이트(소스 전용)가 릴리스될 것으로 예상됩니다.</p>
<h2 id="311의-주요-기능"><a href="#311의-주요-기능">3.11의 주요 기능</a></h2>
<p>Python 3.11의 주목할 만한 기능들은 다음과 같습니다.</p>
<ul>
<li><strong>PEP 654, Exception Groups(예외 그룹) 및 <code>except*</code>:</strong> 여러 예외를 동시에 처리할 수 있는 기능을 제공합니다.</li>
<li><strong>PEP 657, Enhanced error locations in tracebacks (트레이스백의 향상된 오류 위치):</strong> 트레이스백(traceback)에서 오류가 발생한 위치를 더 정확하게 표시하여 디버깅을 돕습니다.</li>
<li><strong>PEP 680, Support for parsing TOML in the standard library (표준 라이브러리에서 TOML 파싱 지원):</strong> 표준 라이브러리에 TOML(Tom's Obvious, Minimal Language) 파일 파싱 지원이 추가됩니다.</li>
<li><strong>성능 향상:</strong> Python 3.11은 Python 3.10보다 최대 10-60% 빠릅니다. 표준 벤치마크 스위트에서 평균 1.25배의 속도 향상이 측정되었습니다. 자세한 내용은 Faster CPython 문서를 참조하십시오.</li>
</ul>
<p><strong>타입 힌트(Typing) 기능:</strong></p>
<ul>
<li><strong>PEP 646, Variadic generics (가변 제네릭):</strong> <code>TypeVarTuple</code>을 사용하여 가변적인 수의 타입 인자를 받을 수 있는 제네릭(Generics)을 정의할 수 있습니다.</li>
<li><strong>PEP 655, Marking individual TypedDict items as required or potentially-missing (개별 TypedDict 항목을 필수 또는 잠재적으로 누락된 것으로 표시):</strong> <code>TypedDict</code>에서 개별 키의 필수 여부를 더욱 명시적으로 지정할 수 있습니다.</li>
<li><strong>PEP 673, Self type (자기 타입):</strong> 클래스 내부에서 자기 자신의 타입을 나타내는 <code>Self</code> 타입을 사용할 수 있습니다.</li>
<li><strong>PEP 675, Arbitrary literal string type (임의의 리터럴 문자열 타입):</strong> 임의의 문자열 리터럴을 타입으로 사용할 수 있게 합니다.</li>
<li><strong>PEP 681, Dataclass transforms (데이터클래스 변환):</strong> <code>dataclass_transform</code> 데코레이터를 통해 사용자 정의 클래스 데코레이터가 <code>dataclass</code>와 유사한 타입 분석 동작을 하도록 만들 수 있습니다.</li>
</ul>
