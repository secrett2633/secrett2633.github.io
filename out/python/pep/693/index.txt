<p><strong>제안 내용:</strong>
PEP 693은 Python 3.12 버전의 개발 및 릴리스 일정에 대한 구체적인 계획을 제공하는 정보성 문서입니다. 여기에는 알파, 베타, 릴리스 후보(Release Candidate) 단계의 시작 및 종료 날짜와 최종 릴리스 날짜가 명시되어 있습니다. 또한, 정기적인 버그 수정 릴리스(bugfix releases)와 최종적으로는 소스 코드 전용(source-only) 보안 수정 릴리스의 일정도 포함되어 있습니다.</p>
<p><strong>도입 배경:</strong>
Python은 PEP 602에 따라 12개월마다 새로운 기능 버전을 릴리스하는 주기(cadence)를 가지고 있습니다. 이러한 정기적인 릴리스 주기를 관리하고, 개발자 및 사용자 커뮤니티가 다음 Python 버전의 출시를 예측하고 준비할 수 있도록 릴리스 스케줄을 명확히 공지하는 것이 중요합니다. 이 PEP는 Python 3.12의 개발 과정과 지원 기간을 투명하게 공유하기 위해 작성되었습니다.</p>
<p><strong>실제 Python 사용에 미치는 영향:</strong></p>
<ol>
<li><strong>예측 가능성:</strong> 개발자들은 이 스케줄을 통해 Python 3.12의 각 개발 단계와 최종 릴리스 시점을 정확히 알 수 있어, 자신의 프로젝트 업그레이드 또는 새로운 프로젝트 시작 시기를 계획할 수 있습니다.</li>
<li><strong>안정성 및 보안:</strong> 정기적인 버그 수정 릴리스와 장기간의 소스 전용 보안 업데이트(2028년 10월까지) 계획은 Python 3.12를 사용하는 애플리케이션의 안정성과 보안을 보장하는 데 기여합니다. 이는 프로덕션 환경에서 Python 3.12를 도입하는 기업이나 개인에게 중요한 요소입니다.</li>
<li><strong>새로운 기능 도입 계획:</strong> "No new features beyond this point"라고 명시된 beta 1 릴리스 시점(2023년 5월 22일)은 개발자들이 3.12에 어떤 기능이 포함될지 최종적으로 확인할 수 있는 시점을 알려줍니다. 이를 통해 개발자들은 자신의 코드 베이스를 3.12의 새로운 기능에 맞춰 준비하거나, 특정 기능이 필요한 경우 릴리스를 기다릴 수 있습니다. "What's New In Python 3.12" 문서를 통해 구체적인 기능 목록을 확인할 수 있습니다.</li>
<li><strong>툴 및 라이브러리 생태계:</strong> Python 3.12의 릴리스 스케줄은 다른 라이브러리 및 툴 개발자들에게도 중요합니다. 이들은 이 스케줄에 맞춰 자신의 제품이 Python 3.12와 호환되도록 업데이트를 계획할 수 있습니다.</li>
</ol>

PEP 693은 Python 3.12 버전의 개발 및 릴리스 일정에 대한 구체적인 계획을 제공하는 정보성 문서입니다. 여기에는 알파, 베타, 릴리스 후보(Release Candidate) 단계의 시작 및 종료 날짜와 최종 릴리스 날짜가 명시되어 있습니다. 또한, 정기적인 버그 수정 릴리스(bugfix releases)와 최종적으로는 소스 코드 전용(source-only) 보안 수정 릴리스의 일정도 포함되어 있습니다.</p>
<p><strong>도입 배경:</strong>
Python은 PEP 602에 따라 12개월마다 새로운 기능 버전을 릴리스하는 주기(cadence)를 가지고 있습니다. 이러한 정기적인 릴리스 주기를 관리하고, 개발자 및 사용자 커뮤니티가 다음 Python 버전의 출시를 예측하고 준비할 수 있도록 릴리스 스케줄을 명확히 공지하는 것이 중요합니다. 이 PEP는 Python 3.12의 개발 과정과 지원 기간을 투명하게 공유하기 위해 작성되었습니다.</p>
<p><strong>실제 Python 사용에 미치는 영향:</strong></p>
<ol>
<li><strong>예측 가능성:</strong> 개발자들은 이 스케줄을 통해 Python 3.12의 각 개발 단계와 최종 릴리스 시점을 정확히 알 수 있어, 자신의 프로젝트 업그레이드 또는 새로운 프로젝트 시작 시기를 계획할 수 있습니다.</li>
<li><strong>안정성 및 보안:</strong> 정기적인 버그 수정 릴리스와 장기간의 소스 전용 보안 업데이트(2028년 10월까지) 계획은 Python 3.12를 사용하는 애플리케이션의 안정성과 보안을 보장하는 데 기여합니다. 이는 프로덕션 환경에서 Python 3.12를 도입하는 기업이나 개인에게 중요한 요소입니다.</li>
<li><strong>새로운 기능 도입 계획:</strong> "No new features beyond this point"라고 명시된 beta 1 릴리스 시점(2023년 5월 22일)은 개발자들이 3.12에 어떤 기능이 포함될지 최종적으로 확인할 수 있는 시점을 알려줍니다. 이를 통해 개발자들은 자신의 코드 베이스를 3.12의 새로운 기능에 맞춰 준비하거나, 특정 기능이 필요한 경우 릴리스를 기다릴 수 있습니다. "What's New In Python 3.12" 문서를 통해 구체적인 기능 목록을 확인할 수 있습니다.</li>
<li><strong>툴 및 라이브러리 생태계:</strong> Python 3.12의 릴리스 스케줄은 다른 라이브러리 및 툴 개발자들에게도 중요합니다. 이들은 이 스케줄에 맞춰 자신의 제품이 Python 3.12와 호환되도록 업데이트를 계획할 수 있습니다.</li>
</ol>
<p>결론적으로, PEP 693은 Python 3.12의 개발 및 지원 일정을 명확히 하여 Python 생태계의 모든 참여자가 계획을 세우고 안정적으로 Python을 활용할 수 있도록 돕는 중요한 정보 문서입니다.</p>
