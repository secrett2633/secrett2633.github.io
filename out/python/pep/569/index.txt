<h1>PEP 569 – Python 3.8 릴리스 스케줄</h1>
<p><strong>작성자:</strong> Łukasz Langa 
<strong>상태:</strong> Final
<strong>유형:</strong> Informational

<strong>Python 버전:</strong> 3.8</p>
<h2>개요 (Abstract)</h2>
<p>이 문서는 Python 3.8의 개발 및 릴리스 스케줄을 설명합니다. 스케줄은 주로 PEP 스케일에 해당하는 항목들을 다룹니다.</p>
<h2>릴리스 관리자 및 담당 팀 (Release Manager and Crew)</h2>
<ul>
<li><strong>3.8 릴리스 관리자:</strong> Łukasz Langa</li>

<li><strong>Mac 인스톨러:</strong> Ned Deily</li>
<li><strong>문서:</strong> Julien Palard</li>
</ul>
<h2>3.8 수명 주기 (3.8 Lifespan)</h2>
<p>Python 3.8은 약 18개월 동안 약 2개월마다 버그 수정 업데이트를 받았습니다. 3.9.0 최종 버전이 릴리스된 후 얼마 지나지 않아 아홉 번째이자 마지막 3.8 버그 수정 업데이트가 릴리스되었습니다. 그 이후에는 Python 3.13.0 최종 버전이 릴리스될 때까지 5년 동안 보안 업데이트(소스 코드만 해당)가 제공되었습니다.</p>
<p>2024년 10월 7일 기준으로, 3.8은 릴리스 주기에서 지원 종료(end-of-life) 단계에 도달했습니다. 3.8.20이 마지막 보안 릴리스였습니다. 3.8의 코드베이스는 이제 고정(frozen)되었으며, 더 이상의 업데이트는 제공되지 않고 버그 트래커에서 어떠한 종류의 문제도 접수되지 않을 것입니다.</p>

<h2>3.8의 주요 기능 (Features for 3.8)</h2>
<p>Python 3.8의 주목할 만한 기능들은 다음과 같습니다.</p>
<ul>
<li><strong>PEP 570, Positional-only arguments (위치 전용 인자):</strong> 함수 인자를 위치로만 전달해야 함을 명시하는 구문입니다.</li>
<li><strong>PEP 572, Assignment Expressions (할당 표현식):</strong> <code>:=</code> 연산자를 사용하여 표현식 내부에서 변수를 할당할 수 있도록 합니다. (일명 "walrus operator")</li>
<li><strong>PEP 574, Pickle protocol 5 with out-of-band data (아웃-오브-밴드 데이터를 사용하는 Pickle 프로토콜 5):</strong> 대량 데이터를 더 효율적으로 직렬화하고 역직렬화할 수 있도록 합니다.</li>
<li><strong>PEP 578, Runtime audit hooks (런타임 감사 훅):</strong> CPython의 런타임 이벤트를 모니터링하고 가로챌 수 있는 API를 제공하여 보안 및 디버깅 기능을 향상시킵니다.</li>
<li><strong>PEP 587, Python Initialization Configuration (Python 초기화 설정):</strong> Python 인터프리터의 초기화 프로세스를 더 세밀하게 제어할 수 있는 API를 제공합니다.</li>
<li><strong>PEP 590, Vectorcall: a fast calling protocol for CPython (Vectorcall: CPython을 위한 빠른 호출 프로토콜):</strong> CPython에서 호출 가능한 객체(callable objects)의 호출 성능을 최적화하는 새로운 내부 프로토콜입니다.</li>
<li><strong>타입 관련 기능:</strong>
<ul>
<li><strong>PEP 591 (Final qualifier):</strong> 변수, 함수, 클래스가 재정의되거나 재할당되지 않음을 나타내는 <code>Final</code> 한정자입니다.</li>
<li><strong>PEP 586 (Literal types):</strong> 특정 리터럴 값만을 허용하는 타입을 정의할 수 있게 합니다.</li>
<li><strong>PEP 589 (TypedDict):</strong> 딕셔너리에 타입 힌트를 적용하여 구조화된 딕셔너리의 키와 값에 대한 타입을 명시할 수 있게 합니다.</li>
</ul>
</li>
<li>컴파일된 바이트코드(bytecode)를 위한 병렬 파일 시스템 캐시.</li>
<li>디버그 빌드가 릴리스 빌드와 ABI를 공유합니다.</li>
<li>f-string에 디버깅을 위한 유용한 <code>=</code> 지정자(specifier)를 지원합니다.
<ul>
<li>예시: <code>f'{variable=}'</code>은 <code>variable=value</code>와 같이 출력됩니다.</li>
</ul>
</li>
<li><code>finally:</code> 블록 내에서 <code>continue</code>가 이제 유효합니다.</li>
<li>Windows에서 기본 <code>asyncio</code> 이벤트 루프(event loop)는 이제 <code>ProactorEventLoop</code>입니다.</li>
<li>macOS에서 <code>multiprocessing</code>의 기본 시작 방식은 이제 <code>spawn</code>입니다.</li>
<li><code>multiprocessing</code>이 이제 공유 메모리(shared memory) 세그먼트를 사용하여 프로세스 간 <code>pickle</code>링 비용을 피할 수 있습니다.</li>
<li><code>typed_ast</code>가 CPython으로 다시 병합되었습니다.</li>
<li><code>LOAD_GLOBAL</code> 연산이 이제 40% 더 빨라졌습니다.</li>
<li><code>pickle</code>이 이제 기본적으로 <code>Protocol 4</code>를 사용하여 성능이 향상되었습니다.</li>
</ul>
<p>이 외에도 많은 흥미로운 변경 사항이 있으며, 전체 목록은 문서의 "새로운 기능(What's New)" 페이지를 참조하시기 바랍니다.</p>
