<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0241/">PEP 241 - Metadata for Python Software Packages</a></p>
<p><strong>상태:</strong> Superseded | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 12-Mar-2001</p>
</blockquote>
<p>PEP 241은 Python 소프트웨어 패키지에 메타데이터를 추가하는 메커니즘을 설명합니다. 이 문서는 필드 이름, 의미 및 사용법에 대한 세부 사항을 포함합니다. 이 PEP는 현재 <strong>Superseded (대체됨)</strong> 상태이며, PyPA (Python Packaging Authority) 스펙 페이지에서 Core metadata specifications (코어 메타데이터 사양)으로 최신 내용이 관리되고 있습니다.</p>
<hr>
<h3><strong>PEP 241 – Python 소프트웨어 패키지용 메타데이터</strong></h3>
<ul>
<li><strong>작성자:</strong> A.M. Kuchling</li>
<li><strong>토론:</strong> Distutils-SIG list</li>
<li><strong>상태:</strong> Superseded (대체됨)</li>
<li><strong>유형:</strong> Standards Track</li>
<li><strong>주제:</strong> Packaging</li>
<li><strong>생성일:</strong> 2001년 3월 12일</li>
<li><strong>대체됨:</strong> PEP 314</li>
</ul>
<h3><strong>개요 (Introduction)</strong></h3>
<p>이 PEP는 Python 패키지에 메타데이터를 추가하는 메커니즘을 설명합니다. 여기에는 필드 이름과 해당 필드의 의미 및 사용법에 대한 세부 정보가 포함됩니다.</p>
<p><strong>중요:</strong> 이 PEP는 역사적인 문서입니다. 최신 규격인 Core metadata specifications는 PyPA 스펙 페이지에서 유지 관리됩니다.</p>
<h3><strong>패키지에 메타데이터 포함 (Including Metadata in Packages)</strong></h3>
<p>Distutils의 <code>sdist</code> 명령어는 인자로부터 메타데이터 필드를 추출하여 생성된 zipfile 또는 tarball 내의 파일에 기록하도록 수정될 것입니다. 이 파일은 <code>PKG-INFO</code>로 명명되며, 소스 배포판의 최상위 디렉토리(일반적으로 <code>README</code>, <code>INSTALL</code> 등의 파일이 위치하는 곳)에 배치됩니다.</p>
<p>개발자는 직접 <code>PKG-INFO</code> 파일을 제공해서는 안 됩니다. <code>sdist</code> 명령어는 기존 <code>PKG-INFO</code> 파일을 감지하면 적절한 오류 메시지와 함께 종료됩니다. 이는 <code>PKG-INFO</code>와 <code>setup.py</code> 파일이 동기화되지 않아 발생하는 혼란을 방지하기 위함입니다.</p>
<p><code>PKG-INFO</code> 파일 형식은 <code>rfc822.py</code> 모듈로 파싱 가능한 단일 RFC 822 헤더 집합입니다. 다음 섹션에 나열된 필드 이름은 헤더 이름으로 사용됩니다. 이 간단한 형식에는 확장 메커니즘이 없으며, Catalog 및 Distutils SIG는 Python 2.2를 위해 더 유연한 형식을 준비하는 것을 목표로 합니다.</p>
<h3><strong>필드 (Fields)</strong></h3>
<p>이 섹션에서는 지원되는 각 메타데이터 필드의 이름과 의미를 지정합니다.</p>
<p>"(Multiple use)"로 표시된 필드는 단일 <code>PKG-INFO</code> 파일에 여러 번 지정될 수 있습니다. 다른 필드는 <code>PKG-INFO</code> 파일에 한 번만 나타날 수 있습니다. "(optional)"로 표시된 필드는 유효한 <code>PKG-INFO</code> 파일에 반드시 나타날 필요는 없으며, 다른 모든 필드는 존재해야 합니다.</p>
<ul>
<li>
<p><strong><code>Metadata-Version</code></strong></p>
<ul>
<li>파일 형식의 버전입니다. 현재 "1.0"만이 유효한 값입니다.</li>
<li>예시: <code>Metadata-Version: 1.0</code></li>
</ul>
</li>
<li>
<p><strong><code>Name</code></strong></p>
<ul>
<li>패키지의 이름입니다.</li>
<li>예시: <code>Name: BeagleVote</code></li>
</ul>
</li>
<li>
<p><strong><code>Version</code></strong></p>
<ul>
<li>패키지의 버전 번호를 포함하는 문자열입니다. 이 필드는 <code>distutils.version</code> 모듈의 <code>Version</code> 클래스(StrictVersion 또는 LooseVersion) 중 하나로 파싱 가능해야 합니다.</li>
<li>예시: <code>Version: 1.0a2</code></li>
</ul>
</li>
<li>
<p><strong><code>Platform</code> (Multiple use)</strong></p>
<ul>
<li>패키지에서 지원하는 운영 체제를 요약한 쉼표로 구분된 플랫폼 사양 목록입니다.</li>
<li>예시: <code>Platform: POSIX, Windows</code></li>
</ul>
</li>
<li>
<p><strong><code>Summary</code></strong></p>
<ul>
<li>패키지가 수행하는 작업을 한 줄로 요약한 설명입니다.</li>
<li>예시: <code>Summary: A module for collecting votes from beagles.</code></li>
</ul>
</li>
<li>
<p><strong><code>Description</code> (optional)</strong></p>
<ul>
<li>여러 단락에 걸쳐 작성될 수 있는 패키지에 대한 더 긴 설명입니다.</li>
<li>예시: <code>Description: This module collects votes from beagles...</code></li>
</ul>
</li>
<li>
<p><strong><code>Keywords</code> (optional)</strong></p>
<ul>
<li>더 큰 카탈로그에서 패키지를 검색하는 데 도움이 되는 추가 키워드 목록입니다.</li>
<li>예시: <code>Keywords: dog puppy voting election</code></li>
</ul>
</li>
<li>
<p><strong><code>Home-page</code> (optional)</strong></p>
<ul>
<li>패키지의 홈 페이지 URL을 포함하는 문자열입니다.</li>
<li>예시: <code>Home-page: http://www.example.com/~cschultz/bvote/</code></li>
</ul>
</li>
<li>
<p><strong><code>Author</code> (optional)</strong></p>
<ul>
<li>최소한 저자의 이름을 포함하는 문자열입니다. 각 줄을 개행 문자로 구분하여 연락처 정보도 추가할 수 있습니다.</li>
<li>예시: <code>Author: C. Schultz Universal Features Syndicate Los Angeles, CA</code></li>
</ul>
</li>
<li>
<p><strong><code>Author-email</code></strong></p>
<ul>
<li>저자의 이메일 주소를 포함하는 문자열입니다. RFC 822 'From:' 헤더의 유효한 형식으로 이름과 이메일 주소를 포함할 수 있습니다. 카탈로그 시스템에서 이 필드의 이메일 부분을 저자를 나타내는 고유 키로 사용할 수 있으므로 필수는 아닙니다.</li>
<li>예시: <code>Author-email: "C. Schultz" &#x3C;cschultz@example.com></code></li>
</ul>
</li>
<li>
<p><strong><code>License</code></strong></p>
<ul>
<li>패키지를 다루는 라이선스를 지정하는 짧은 선택 목록에서 선택된 문자열입니다. 일부 라이선스는 소프트웨어가 자유롭게 재배포될 수 있음을 의미하므로, 패키저와 리셀러는 소프트웨어를 자유롭게 재배포할 수 있음을 자동으로 알 수 있습니다.</li>
<li>선택 가능한 라이선스 목록: Artistic, BSD, DFSG, GNU GPL, GNU LGPL, "MIT", Mozilla PL, "public domain", Python, Qt PL, Zope PL, unknown, nocommercial, nosell, nosource, shareware, other.</li>
<li>예시: <code>License: MIT</code></li>
</ul>
</li>
</ul>
