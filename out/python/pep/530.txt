<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0530/" target="_blank" rel="noopener noreferrer">PEP 530 - Asynchronous Comprehensions</a></p>

<h2 id="pep-530-비동기-컴프리헨션-asynchronous-comprehensions"><a href="#pep-530-비동기-컴프리헨션-asynchronous-comprehensions">PEP 530 – 비동기 컴프리헨션 (Asynchronous Comprehensions)</a></h2>
<h3 id="개요"><a href="#개요">개요</a></h3>
<p>PEP 530은 Python 3.6에 도입된 기능으로, <code>async</code>/<code>await</code> 문법을 사용하는 비동기 버전의 <code>list</code>, <code>set</code>, <code>dict</code> 컴프리헨션(comprehension) 및 제너레이터 표현식(generator expressions)을 추가할 것을 제안합니다. 이는 PEP 492 (네이티브 코루틴)와 PEP 525 (비동기 제너레이터)에서 제공하는 비동기 프로그래밍 지원을 확장하는 것입니다.</p>
<h3 id="배경-및-목표"><a href="#배경-및-목표">배경 및 목표</a></h3>
<p>Python은 동기 컴프리헨션에 대한 광범위한 지원을 제공하여 간결한 문법으로 <code>list</code>, <code>dict</code>, <code>set</code>을 생성할 수 있게 합니다. PEP 530은 이러한 구문적 구성을 비동기 코드에도 적용하여 가독성을 높이고 코드를 단축하는 것을 목표로 합니다.</p>
<p><strong>가독성 향상 예시:</strong></p>
<p>기존 비동기 코드는 다음과 같을 수 있습니다.</p>
<pre><code class="language-python">result = []
async for i in aiter():
    if i % 2:
        result.append(i)
</code></pre>
<p>PEP 530이 제안하는 비동기 컴프리헨션 문법을 사용하면 위 코드는 다음과 같이 간결해집니다.</p>
<pre><code class="language-python">result = [i async for i in aiter() if i % 2]
</code></pre>
<p>또한, 이 PEP는 모든 종류의 컴프리헨션 내에서 <code>await</code> 표현식을 사용할 수 있도록 합니다. 예를 들어:</p>
<pre><code class="language-python">result = [await fun() for fun in funcs]
</code></pre>
<h3 id="상세-내용-specification"><a href="#상세-내용-specification">상세 내용 (Specification)</a></h3>
<h4 id="비동기-컴프리헨션-asynchronous-comprehensions"><a href="#비동기-컴프리헨션-asynchronous-comprehensions">비동기 컴프리헨션 (Asynchronous Comprehensions)</a></h4>
<p><code>list</code>, <code>set</code>, <code>dict</code> 컴프리헨션 내에서 <code>async for</code>를 사용하는 것이 허용됩니다. PEP 525의 승인에 따라 비동기 제너레이터 표현식 생성도 가능해집니다.</p>
<p><strong>예시:</strong></p>
<ul>
<li><code>set</code> 컴프리헨션: <code>{i async for i in agen()}</code></li>
<li><code>list</code> 컴프리헨션: <code>[i async for i in agen()]</code></li>
<li><code>dict</code> 컴프리헨션: <code>{i: i ** 2 async for i in agen()}</code></li>
<li>제너레이터 표현식: <code>(i ** 2 async for i in agen())</code></li>
</ul>
<p>비동기 컴프리헨션 및 제너레이터 표현식 내에서 <code>async for</code>는 <code>if</code> 및 <code>for</code> 절과 함께 사용될 수 있습니다.</p>
<pre><code class="language-python">dataset = {data for line in aiter() async for data in line if check(data)}
</code></pre>
<p>비동기 컴프리헨션은 오직 <code>async def</code> 함수 내부에서만 허용됩니다.
비동기 제너레이터 표현식은 일반적으로 어떤 문맥에서든 허용되지만, Python 3.6에서는 <code>async</code>와 <code>await</code>가 소프트 키워드(soft-keyword) 상태였기 때문에 <code>async def</code> 함수 내에서만 허용되었습니다. Python 3.7에서 <code>async</code>와 <code>await</code>가 예약어(reserved keywords)가 되면서 이 제한은 해제됩니다.</p>
<h4 id="컴프리헨션-내에서-await-사용-await-in-comprehensions"><a href="#컴프리헨션-내에서-await-사용-await-in-comprehensions">컴프리헨션 내에서 <code>await</code> 사용 (await in Comprehensions)</a></h4>
<p>비동기 컴프리헨션뿐만 아니라 동기 컴프리헨션 내에서도 <code>await</code> 표현식의 사용이 허용됩니다.
이 기능은 <code>async def</code> 함수 본문 내에서만 유효합니다.</p>
<p>** 예시: **</p>
<pre><code class="language-python">result = [await fun() for fun in funcs]
result = {await fun() for fun in funcs}
result = {fun: await fun() for fun in funcs}
result = [await fun() for fun in funcs if await smth]
result = {await fun() for fun in funcs if await smth}
result = {fun: await fun() for fun in funcs if await smth}
result = [await fun() async for fun in funcs]
result = {await fun() async for fun in funcs}
result = {fun: await fun() async for fun in funcs}
result = [await fun() async for fun in funcs if await smth]
result = {await fun() async for fun in funcs if await smth}
result = {fun: await fun() async for fun in funcs if await smth}
</code></pre>
<h4 id="문법-업데이트-grammar-updates"><a href="#문법-업데이트-grammar-updates">문법 업데이트 (Grammar Updates)</a></h4>
<p>이 제안은 문법 수준에서 <code>comp_for</code> 규칙에 선택적 <code>ASYNC</code> 키워드를 추가하는 한 가지 변경을 요구합니다.
<code>comp_for: [ASYNC] 'for' exprlist 'in' or_test [comp_iter]</code></p>
<p>컴프리헨션 AST(Abstract Syntax Tree) 노드에는 새로운 <code>is_async</code> 인자가 추가됩니다.</p>
<h3 id="하위-호환성-backwards-compatibility"><a href="#하위-호환성-backwards-compatibility">하위 호환성 (Backwards Compatibility)</a></h3>
