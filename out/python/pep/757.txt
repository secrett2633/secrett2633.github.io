<h1 id="pep-757-c-api를-이용한-python-정수-importexport"><a href="#pep-757-c-api를-이용한-python-정수-importexport">PEP 757 – C API를 이용한 Python 정수 import/export</a></h1>
<h2 id="개요"><a href="#개요">개요</a></h2>
<p>새로운 C API, 특히 <code>PyLongWriter_Create()</code> 및 <code>PyLong_Export()</code> 함수를 추가하여 Python 정수(<code>int</code> 객체)를 import하고 export하는 기능을 제공합니다.</p>
<h2 id="도입-배경-rationale"><a href="#도입-배경-rationale">도입 배경 (Rationale)</a></h2>
<p><code>gmpy2</code>, <code>SAGE</code>, <code>Python-FLINT</code>와 같은 프로젝트들은 Python의 내부(<code>PyLongObject</code> 구조체)에 직접 접근하거나 비효율적인 임시 형식(예: <code>Python-FLINT</code>의 16진수 문자열)을 사용하여 Python <code>int</code> 객체를 import 및 export합니다.</p>
<p>Python 3.12에서는 <code>int</code> 구현이 <code>tag</code>와 "compact values"를 추가하는 방향으로 변경되었습니다. 3.13 alpha 1 릴리스에서는 비공개, 비문서화된 <code>_PyLong_New()</code> 함수가 제거되었지만, 이 함수는 앞서 언급된 프로젝트들에서 Python 정수를 import하는 데 사용되고 있었습니다. 이 함수는 3.13 alpha 2에서 복원되었습니다.</p>
<p>이러한 프로젝트들이 Python의 내부 구현 세부 사항에 노출되지 않으면서도 효율적으로 연동될 수 있도록, 공개적이고 효율적인 추상화 계층이 필요합니다. 이는 Python 내부 구현이 변경되더라도 해당 프로젝트들이 깨지지 않도록 할 것입니다. 예를 들어, <code>gmpy2</code>의 구현은 최근 CPython 3.9 및 CPython 3.12에 맞춰 변경된 바 있습니다.</p>
<h2 id="명세-specification"><a href="#명세-specification">명세 (Specification)</a></h2>
<h3 id="레이아웃-api-layout-api"><a href="#레이아웃-api-layout-api">레이아웃 API (Layout API)</a></h3>
<p>GMP(GNU Multiple Precision Arithmetic Library)와 유사한 import-export 함수에 필요한 데이터입니다.</p>
<h4 id="struct-pylonglayout"><a href="#struct-pylonglayout"><code>struct PyLongLayout</code></a></h4>
<p>임의 정밀도 정수의 절대값을 나타내는 "digit" 배열(GMP 용어로는 "limb")의 레이아웃입니다. "충분히 큰" 절대값을 가진 정수에서 내부적으로 사용되는 Python <code>int</code> 객체의 기본 레이아웃을 얻으려면 <code>PyLong_GetNativeLayout()</code> 함수를 사용합니다.</p>
<p><code>sys.int_info</code>는 Python에 유사한 정보를 노출합니다.</p>
<ul>
<li><strong><code>uint8_t bits_per_digit</code></strong> : digit당 비트 수. 예를 들어, 15비트 digit은 비트 0-14에 유의미한 정보가 포함됨을 의미합니다.</li>
<li><strong><code>uint8_t digit_size</code></strong> : digit의 바이트 크기. 예를 들어, 15비트 digit은 최소 2바이트를 필요로 합니다.</li>
<li><strong><code>int8_t digits_order</code></strong> : digit 순서:
<ul>
<li><code>1</code>: 최상위 digit 우선 (most significant digit first)</li>
<li><code>-1</code>: 최하위 digit 우선 (least significant digit first)</li>
</ul>
</li>
<li><strong><code>int8_t digit_endianness</code></strong> : digit의 엔디안(endianness):
<ul>
<li><code>1</code>: 최상위 바이트 우선 (big endian)</li>
<li><code>-1</code>: 최하위 바이트 우선 (little endian)</li>
</ul>
</li>
</ul>
<h4 id="const-pylonglayout-pylong_getnativelayoutvoid"><a href="#const-pylonglayout-pylong_getnativelayoutvoid"><code>const PyLongLayout *PyLong_GetNativeLayout(void)</code></a></h4>
<p>Python <code>int</code> 객체의 기본 레이아웃을 가져옵니다. <code>PyLongLayout</code> 구조체를 참조하세요.
이 함수는 Python 초기화 전이나 종료 후에는 호출되어서는 안 됩니다. 반환된 레이아웃은 Python이 종료될 때까지 유효합니다. 이 레이아웃은 모든 Python 서브 인터프리터에서 동일하므로 캐시할 수 있습니다.</p>
<h3 id="export-api"><a href="#export-api">Export API</a></h3>
<p>Python <code>int</code> 객체를 export하는 데 사용됩니다.</p>
<h4 id="struct-pylongexport"><a href="#struct-pylongexport"><code>struct PyLongExport</code></a></h4>
<p>Python <code>int</code> 객체의 export 구조입니다. 두 가지 경우가 있습니다:</p>
<ul>
<li>
<p><code>digits</code>가 <code>NULL</code>인 경우, <code>value</code> 멤버만 사용합니다.</p>
</li>
<li>
<p><code>digits</code>가 <code>NULL</code>이 아닌 경우, <code>negative</code>, <code>ndigits</code>, <code>digits</code> 멤버를 사용합니다.</p>
</li>
<li>
<p><strong><code>int64_t value</code></strong> : export된 <code>int</code> 객체의 네이티브 정수 값. <code>digits</code>가 <code>NULL</code>인 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>uint8_t negative</code></strong> : 숫자가 음수이면 <code>1</code>, 아니면 <code>0</code>. <code>digits</code>가 <code>NULL</code>이 아닌 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>Py_ssize_t ndigits</code></strong> : <code>digits</code> 배열의 digit 수. <code>digits</code>가 <code>NULL</code>이 아닌 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>const void *digits</code></strong>: 읽기 전용 부호 없는 digit 배열. <code>NULL</code>일 수 있습니다.</p>
</li>
</ul>
<p><code>PyLongExport.digits</code>가 <code>NULL</code>이 아닌 경우, <code>PyLongExport</code> 구조체의 private 필드에 Python <code>int</code> 객체에 대한 강력한 참조(strong reference)를 저장하여 <code>PyLong_FreeExport()</code>가 호출될 때까지 구조체가 유효하게 유지되도록 합니다.</p>
<h4 id="int-pylong_exportpyobject-obj-pylongexport-export_long"><a href="#int-pylong_exportpyobject-obj-pylongexport-export_long"><code>int PyLong_Export(PyObject *obj, PyLongExport *export_long)</code></a></h4>
<p>Python <code>int</code> 객체를 export합니다.
<code>export_long</code>은 호출자가 할당한 <code>PyLongExport</code> 구조체를 가리켜야 하며, <code>NULL</code>이 아니어야 합니다.
성공 시 <code>*export_long</code>을 채우고 <code>0</code>을 반환합니다. 오류 발생 시 예외를 설정하고 <code>-1</code>을 반환합니다.
export가 더 이상 필요하지 않을 때 <code>PyLong_FreeExport()</code>를 호출해야 합니다.</p>
<p>CPython 구현 세부 사항: <code>obj</code>가 Python <code>int</code> 객체 또는 서브클래스인 경우 이 함수는 항상 성공합니다.
CPython 3.14에서는 <code>PyLong_Export()</code>에서 메모리 복사가 필요하지 않으며, 이는 Python <code>int</code>의 내부 digit 배열을 노출하기 위한 얇은 래퍼(thin wrapper)에 불과합니다.</p>
<h4 id="void-pylong_freeexportpylongexport-export_long"><a href="#void-pylong_freeexportpylongexport-export_long"><code>void PyLong_FreeExport(PyLongExport *export_long)</code></a></h4>
<p><code>PyLong_Export()</code>로 생성된 <code>export_long</code> export를 해제합니다.
CPython 구현 세부 사항: <code>export_long->digits</code>가 <code>NULL</code>인 경우 <code>PyLong_FreeExport()</code> 호출은 선택 사항입니다.</p>
<h3 id="import-api"><a href="#import-api">Import API</a></h3>
<p><code>PyLongWriter</code> API를 사용하여 정수를 import할 수 있습니다.</p>
<h4 id="struct-pylongwriter"><a href="#struct-pylongwriter"><code>struct PyLongWriter</code></a></h4>
<p>Python <code>int</code> writer 인스턴스입니다.
이 인스턴스는 <code>PyLongWriter_Finish()</code> 또는 <code>PyLongWriter_Discard()</code>에 의해 파괴되어야 합니다.</p>
<h4 id="pylongwriter-pylongwriter_createint-negative-py_ssize_t-ndigits-void-digits"><a href="#pylongwriter-pylongwriter_createint-negative-py_ssize_t-ndigits-void-digits"><code>PyLongWriter *PyLongWriter_Create(int negative, Py_ssize_t ndigits, void **digits)</code></a></h4>
<p><code>PyLongWriter</code>를 생성합니다.
성공 시 <code>*digits</code>를 할당하고 writer를 반환합니다. 오류 발생 시 예외를 설정하고 <code>NULL</code>을 반환합니다.
<code>negative</code>는 숫자가 음수이면 <code>1</code>, 아니면 <code>0</code>입니다.
<code>ndigits</code>는 <code>digits</code> 배열의 숫자(digit) 수입니다. <code>0</code>보다 커야 합니다.
<code>digits</code>는 <code>NULL</code>이 아니어야 합니다.</p>
<p>이 함수를 성공적으로 호출한 후, 호출자는 <code>digits</code> 배열을 채우고 <code>PyLongWriter_Finish()</code>를 호출하여 Python <code>int</code>를 얻어야 합니다. <code>digits</code>의 레이아웃은 <code>PyLong_GetNativeLayout()</code>에 의해 설명됩니다.
Digits는 <code>[0; (1 &#x3C;&#x3C; bits_per_digit) - 1]</code> 범위에 있어야 합니다 (<code>bits_per_digit</code>는 digit당 비트 수). 사용되지 않는 최상위 digit은 <code>0</code>으로 설정해야 합니다.
또는 <code>PyLongWriter_Discard()</code>를 호출하여 <code>int</code> 객체를 생성하지 않고 writer 인스턴스를 파괴할 수 있습니다.</p>
<p>CPython 3.14에서는 <code>PyLongWriter_Create()</code> 구현이 private <code>_PyLong_New()</code> 함수에 대한 얇은 래퍼입니다.</p>
<h4 id="pyobject-pylongwriter_finishpylongwriter-writer"><a href="#pyobject-pylongwriter_finishpylongwriter-writer"><code>PyObject *PyLongWriter_Finish(PyLongWriter *writer)</code></a></h4>
<p><code>PyLongWriter_Create()</code>로 생성된 <code>PyLongWriter</code>를 완료합니다.
성공 시 Python <code>int</code> 객체를 반환합니다. 오류 발생 시 예외를 설정하고 <code>NULL</code>을 반환합니다.
이 함수는 digit을 정규화하고 필요한 경우 객체를 compact integer로 변환합니다.
호출 후 writer 인스턴스와 digits 배열은 유효하지 않습니다.</p>
<h4 id="void-pylongwriter_discardpylongwriter-writer"><a href="#void-pylongwriter_discardpylongwriter-writer"><code>void PyLongWriter_Discard(PyLongWriter *writer)</code></a></h4>
<p><code>PyLongWriter_Create()</code>로 생성된 <code>PyLongWriter</code>를 폐기합니다.
<code>writer</code>는 <code>NULL</code>이 아니어야 합니다.
호출 후 writer 인스턴스와 digits 배열은 유효하지 않습니다.</p>
<h3 id="작은-정수-import-최적화-optimize-import-for-small-integers"><a href="#작은-정수-import-최적화-optimize-import-for-small-integers">작은 정수 import 최적화 (Optimize import for small integers)</a></h3>
<p>제안된 import API는 큰 정수에 효율적입니다. Python 내부(<code>internals</code>)에 직접 접근하는 것과 비교할 때, 제안된 import API는 작은 정수에서 상당한 성능 오버헤드를 가질 수 있습니다.</p>
<p>몇 개의 digit을 가진 작은 정수(예: 1 또는 2 digit)의 경우, 기존 API를 사용할 수 있습니다:</p>
<ul>
<li><code>PyLong_FromUInt64()</code></li>
<li><code>PyLong_FromLong()</code></li>
<li><code>PyLong_FromNativeBytes()</code></li>
</ul>

<h2 id="하위-호환성-backwards-compatibility"><a href="#하위-호환성-backwards-compatibility">하위 호환성 (Backwards Compatibility)</a></h2>
<p>새로운 API만 추가되므로 하위 호환성에 미치는 영향은 없습니다.</p>
<h2 id="기각된-아이디어-rejected-ideas"><a href="#기각된-아이디어-rejected-ideas">기각된 아이디어 (Rejected Ideas)</a></h2>
<h3 id="임의의-레이아웃-지원-support-arbitrary-layout"><a href="#임의의-레이아웃-지원-support-arbitrary-layout">임의의 레이아웃 지원 (Support arbitrary layout)</a></h3>
<p>Python 정수를 import-export하기 위해 임의의 레이아웃을 지원하는 것은 편리할 수 있습니다. 예를 들어, <code>PyLongWriter_Create()</code>에 <code>layout</code> 매개변수를 추가하고 <code>PyLongExport</code> 구조체에 <code>layout</code> 멤버를 추가하는 것이 제안되었습니다.
그러나 구현이 더 복잡하고 실제로 필요하지 않습니다. 엄밀히 필요한 것은 Python의 "네이티브(native)" 레이아웃을 사용하여 import-export하는 API뿐입니다.
나중에 임의의 레이아웃에 대한 사용 사례가 있다면 새로운 API를 추가할 수 있습니다.</p>
<h3 id="pylong_getnativelayout-함수를-추가하지-않음-dont-add-pylong_getnativelayout-function"><a href="#pylong_getnativelayout-함수를-추가하지-않음-dont-add-pylong_getnativelayout-function"><code>PyLong_GetNativeLayout()</code> 함수를 추가하지 않음 (Don't add PyLong_GetNativeLayout() function)</a></h3>
<p>현재 대부분의 정수 import/export에 필요한 정보는 <code>PyLong_GetInfo()</code> (및 <code>sys.int_info</code>)를 통해 이미 사용 가능합니다. 또한, 더 많은 정보(예: digits 순서)를 추가할 수 있으며, 이 인터페이스는 <code>PyLongObject</code>의 미래 발전에 어떤 제약도 가하지 않습니다.
문제는 <code>PyLong_GetInfo()</code>가 편리한 C 구조체가 아닌 Python 객체(named tuple)를 반환한다는 점이며, 이는 사람들이 현재의 세미-프라이빗 매크로(<code>PyLong_SHIFT</code> 및 <code>PyLong_BASE</code> 등)를 선호하게 만들 수 있습니다.</p>
<h3 id="mpz_importexport와-유사한-api-제공-provide-mpz_importexport-like-api-instead"><a href="#mpz_importexport와-유사한-api-제공-provide-mpz_importexport-like-api-instead"><code>mpz_import/export</code>와 유사한 API 제공 (Provide mpz_import/export-like API instead)</a></h3>
<p><code>int</code> 객체에서 데이터를 import/export하는 또 다른 접근 방식은 C 확장이 연속적인 버퍼를 제공하고 CPython이 정수의 절대값을 export(또는 import)하는 것을 기대하는 것입니다.
이 접근 방식의 주요 단점은 CPython 측면에서 훨씬 더 복잡하다는 것입니다(즉, 다른 레이아웃 간의 실제 변환). 예를 들어, <code>CPython</code>에서 <code>PyLong_FromNativeBytes()</code> 및 <code>PyLong_AsNativeBytes()</code> 구현(함께 필요한 API의 제한된 버전을 제공)은 약 500줄의 코드를 차지했습니다(현재 구현은 약 100줄).</p>
<h3 id="export-api에서-value-필드를-삭제-drop-value-field-from-the-export-api"><a href="#export-api에서-value-필드를-삭제-drop-value-field-from-the-export-api">export API에서 <code>value</code> 필드를 삭제 (Drop value field from the export API)</a></h3>
<p>이 제안에 따르면, 하나의 export 유형(digit 배열)만 존재하게 됩니다. 주어진 정수에 대해 이러한 뷰를 사용할 수 없는 경우, export 함수에 의해 에뮬레이션되거나 <code>PyLong_Export()</code>가 오류를 반환할 것입니다. 두 경우 모두, 사용자는 <code>PyLong_AsLongAndOverflow()</code>와 같이 "충분히 작은" 정수(즉, 일부 머신 정수 형식에 맞는 정수)를 얻기 위해 다른 C-API 함수를 사용할 것으로 가정합니다. 이 경우 <code>PyLong_Export()</code>는 비효율적이거나 실패할 것입니다.
이는 API 설계자 관점에서는 단순화처럼 보일 수 있지만, 최종 사용자에게는 덜 편리할 것입니다. 사용자는 Python 개발을 따라야 하고, 작은 정수를 export하기 위한 다양한 변형을 벤치마크하며(위의 경우 <code>PyLong_AsInt64()</code> 대신 선택된 이유가 명확한가요?), 다양한 CPython 버전 또는 다른 Python 구현에 대해 다른 코드 경로를 지원해야 할 수도 있습니다.</p>

<h1 id="pep-757-c-api를-이용한-python-정수-importexport"><a href="#pep-757-c-api를-이용한-python-정수-importexport">PEP 757 – C API를 이용한 Python 정수 import/export</a></h1>
<h2 id="개요"><a href="#개요">개요</a></h2>
<p>새로운 C API, 특히 <code>PyLongWriter_Create()</code> 및 <code>PyLong_Export()</code> 함수를 추가하여 Python 정수(<code>int</code> 객체)를 import하고 export하는 기능을 제공합니다.</p>
<h2 id="도입-배경-rationale"><a href="#도입-배경-rationale">도입 배경 (Rationale)</a></h2>
<p><code>gmpy2</code>, <code>SAGE</code>, <code>Python-FLINT</code>와 같은 프로젝트들은 Python의 내부(<code>PyLongObject</code> 구조체)에 직접 접근하거나 비효율적인 임시 형식(예: <code>Python-FLINT</code>의 16진수 문자열)을 사용하여 Python <code>int</code> 객체를 import 및 export합니다.</p>
<p>Python 3.12에서는 <code>int</code> 구현이 <code>tag</code>와 "compact values"를 추가하는 방향으로 변경되었습니다. 3.13 alpha 1 릴리스에서는 비공개, 비문서화된 <code>_PyLong_New()</code> 함수가 제거되었지만, 이 함수는 앞서 언급된 프로젝트들에서 Python 정수를 import하는 데 사용되고 있었습니다. 이 함수는 3.13 alpha 2에서 복원되었습니다.</p>
<p>이러한 프로젝트들이 Python의 내부 구현 세부 사항에 노출되지 않으면서도 효율적으로 연동될 수 있도록, 공개적이고 효율적인 추상화 계층이 필요합니다. 이는 Python 내부 구현이 변경되더라도 해당 프로젝트들이 깨지지 않도록 할 것입니다. 예를 들어, <code>gmpy2</code>의 구현은 최근 CPython 3.9 및 CPython 3.12에 맞춰 변경된 바 있습니다.</p>
<h2 id="명세-specification"><a href="#명세-specification">명세 (Specification)</a></h2>
<h3 id="레이아웃-api-layout-api"><a href="#레이아웃-api-layout-api">레이아웃 API (Layout API)</a></h3>
<p>GMP(GNU Multiple Precision Arithmetic Library)와 유사한 import-export 함수에 필요한 데이터입니다.</p>
<h4 id="struct-pylonglayout"><a href="#struct-pylonglayout"><code>struct PyLongLayout</code></a></h4>
<p>임의 정밀도 정수의 절대값을 나타내는 "digit" 배열(GMP 용어로는 "limb")의 레이아웃입니다. "충분히 큰" 절대값을 가진 정수에서 내부적으로 사용되는 Python <code>int</code> 객체의 기본 레이아웃을 얻으려면 <code>PyLong_GetNativeLayout()</code> 함수를 사용합니다.</p>
<p><code>sys.int_info</code>는 Python에 유사한 정보를 노출합니다.</p>
<ul>
<li><strong><code>uint8_t bits_per_digit</code></strong> : digit당 비트 수. 예를 들어, 15비트 digit은 비트 0-14에 유의미한 정보가 포함됨을 의미합니다.</li>
<li><strong><code>uint8_t digit_size</code></strong> : digit의 바이트 크기. 예를 들어, 15비트 digit은 최소 2바이트를 필요로 합니다.</li>
<li><strong><code>int8_t digits_order</code></strong> : digit 순서:
<ul>
<li><code>1</code>: 최상위 digit 우선 (most significant digit first)</li>
<li><code>-1</code>: 최하위 digit 우선 (least significant digit first)</li>
</ul>
</li>
<li><strong><code>int8_t digit_endianness</code></strong> : digit의 엔디안(endianness):
<ul>
<li><code>1</code>: 최상위 바이트 우선 (big endian)</li>
<li><code>-1</code>: 최하위 바이트 우선 (little endian)</li>
</ul>
</li>
</ul>
<h4 id="const-pylonglayout-pylong_getnativelayoutvoid"><a href="#const-pylonglayout-pylong_getnativelayoutvoid"><code>const PyLongLayout *PyLong_GetNativeLayout(void)</code></a></h4>
<p>Python <code>int</code> 객체의 기본 레이아웃을 가져옵니다. <code>PyLongLayout</code> 구조체를 참조하세요.
이 함수는 Python 초기화 전이나 종료 후에는 호출되어서는 안 됩니다. 반환된 레이아웃은 Python이 종료될 때까지 유효합니다. 이 레이아웃은 모든 Python 서브 인터프리터에서 동일하므로 캐시할 수 있습니다.</p>
<h3 id="export-api"><a href="#export-api">Export API</a></h3>
<p>Python <code>int</code> 객체를 export하는 데 사용됩니다.</p>
<h4 id="struct-pylongexport"><a href="#struct-pylongexport"><code>struct PyLongExport</code></a></h4>
<p>Python <code>int</code> 객체의 export 구조입니다. 두 가지 경우가 있습니다:</p>
<ul>
<li>
<p><code>digits</code>가 <code>NULL</code>인 경우, <code>value</code> 멤버만 사용합니다.</p>
</li>
<li>
<p><code>digits</code>가 <code>NULL</code>이 아닌 경우, <code>negative</code>, <code>ndigits</code>, <code>digits</code> 멤버를 사용합니다.</p>
</li>
<li>
<p><strong><code>int64_t value</code></strong> : export된 <code>int</code> 객체의 네이티브 정수 값. <code>digits</code>가 <code>NULL</code>인 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>uint8_t negative</code></strong> : 숫자가 음수이면 <code>1</code>, 아니면 <code>0</code>. <code>digits</code>가 <code>NULL</code>이 아닌 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>Py_ssize_t ndigits</code></strong> : <code>digits</code> 배열의 digit 수. <code>digits</code>가 <code>NULL</code>이 아닌 경우에만 유효합니다.</p>
</li>
<li>
<p><strong><code>const void *digits</code></strong>: 읽기 전용 부호 없는 digit 배열. <code>NULL</code>일 수 있습니다.</p>
</li>
</ul>
<p><code>PyLongExport.digits</code>가 <code>NULL</code>이 아닌 경우, <code>PyLongExport</code> 구조체의 private 필드에 Python <code>int</code> 객체에 대한 강력한 참조(strong reference)를 저장하여 <code>PyLong_FreeExport()</code>가 호출될 때까지 구조체가 유효하게 유지되도록 합니다.</p>
<h4 id="int-pylong_exportpyobject-obj-pylongexport-export_long"><a href="#int-pylong_exportpyobject-obj-pylongexport-export_long"><code>int PyLong_Export(PyObject *obj, PyLongExport *export_long)</code></a></h4>
<p>Python <code>int</code> 객체를 export합니다.
<code>export_long</code>은 호출자가 할당한 <code>PyLongExport</code> 구조체를 가리켜야 하며, <code>NULL</code>이 아니어야 합니다.
성공 시 <code>*export_long</code>을 채우고 <code>0</code>을 반환합니다. 오류 발생 시 예외를 설정하고 <code>-1</code>을 반환합니다.
export가 더 이상 필요하지 않을 때 <code>PyLong_FreeExport()</code>를 호출해야 합니다.</p>
<p>CPython 구현 세부 사항: <code>obj</code>가 Python <code>int</code> 객체 또는 서브클래스인 경우 이 함수는 항상 성공합니다.
CPython 3.14에서는 <code>PyLong_Export()</code>에서 메모리 복사가 필요하지 않으며, 이는 Python <code>int</code>의 내부 digit 배열을 노출하기 위한 얇은 래퍼(thin wrapper)에 불과합니다.</p>
<h4 id="void-pylong_freeexportpylongexport-export_long"><a href="#void-pylong_freeexportpylongexport-export_long"><code>void PyLong_FreeExport(PyLongExport *export_long)</code></a></h4>
<p><code>PyLong_Export()</code>로 생성된 <code>export_long</code> export를 해제합니다.
CPython 구현 세부 사항: <code>export_long->digits</code>가 <code>NULL</code>인 경우 <code>PyLong_FreeExport()</code> 호출은 선택 사항입니다.</p>
<h3 id="import-api"><a href="#import-api">Import API</a></h3>
<p><code>PyLongWriter</code> API를 사용하여 정수를 import할 수 있습니다.</p>
<h4 id="struct-pylongwriter"><a href="#struct-pylongwriter"><code>struct PyLongWriter</code></a></h4>
<p>Python <code>int</code> writer 인스턴스입니다.
이 인스턴스는 <code>PyLongWriter_Finish()</code> 또는 <code>PyLongWriter_Discard()</code>에 의해 파괴되어야 합니다.</p>
<h4 id="pylongwriter-pylongwriter_createint-negative-py_ssize_t-ndigits-void-digits"><a href="#pylongwriter-pylongwriter_createint-negative-py_ssize_t-ndigits-void-digits"><code>PyLongWriter *PyLongWriter_Create(int negative, Py_ssize_t ndigits, void **digits)</code></a></h4>
<p><code>PyLongWriter</code>를 생성합니다.
성공 시 <code>*digits</code>를 할당하고 writer를 반환합니다. 오류 발생 시 예외를 설정하고 <code>NULL</code>을 반환합니다.
<code>negative</code>는 숫자가 음수이면 <code>1</code>, 아니면 <code>0</code>입니다.
<code>ndigits</code>는 <code>digits</code> 배열의 숫자(digit) 수입니다. <code>0</code>보다 커야 합니다.
<code>digits</code>는 <code>NULL</code>이 아니어야 합니다.</p>
<p>이 함수를 성공적으로 호출한 후, 호출자는 <code>digits</code> 배열을 채우고 <code>PyLongWriter_Finish()</code>를 호출하여 Python <code>int</code>를 얻어야 합니다. <code>digits</code>의 레이아웃은 <code>PyLong_GetNativeLayout()</code>에 의해 설명됩니다.
Digits는 <code>[0; (1 &#x3C;&#x3C; bits_per_digit) - 1]</code> 범위에 있어야 합니다 (<code>bits_per_digit</code>는 digit당 비트 수). 사용되지 않는 최상위 digit은 <code>0</code>으로 설정해야 합니다.
또는 <code>PyLongWriter_Discard()</code>를 호출하여 <code>int</code> 객체를 생성하지 않고 writer 인스턴스를 파괴할 수 있습니다.</p>
<p>CPython 3.14에서는 <code>PyLongWriter_Create()</code> 구현이 private <code>_PyLong_New()</code> 함수에 대한 얇은 래퍼입니다.</p>
<h4 id="pyobject-pylongwriter_finishpylongwriter-writer"><a href="#pyobject-pylongwriter_finishpylongwriter-writer"><code>PyObject *PyLongWriter_Finish(PyLongWriter *writer)</code></a></h4>
<p><code>PyLongWriter_Create()</code>로 생성된 <code>PyLongWriter</code>를 완료합니다.
성공 시 Python <code>int</code> 객체를 반환합니다. 오류 발생 시 예외를 설정하고 <code>NULL</code>을 반환합니다.
이 함수는 digit을 정규화하고 필요한 경우 객체를 compact integer로 변환합니다.
호출 후 writer 인스턴스와 digits 배열은 유효하지 않습니다.</p>
<h4 id="void-pylongwriter_discardpylongwriter-writer"><a href="#void-pylongwriter_discardpylongwriter-writer"><code>void PyLongWriter_Discard(PyLongWriter *writer)</code></a></h4>
<p><code>PyLongWriter_Create()</code>로 생성된 <code>PyLongWriter</code>를 폐기합니다.
<code>writer</code>는 <code>NULL</code>이 아니어야 합니다.
호출 후 writer 인스턴스와 digits 배열은 유효하지 않습니다.</p>
<h3 id="작은-정수-import-최적화-optimize-import-for-small-integers"><a href="#작은-정수-import-최적화-optimize-import-for-small-integers">작은 정수 import 최적화 (Optimize import for small integers)</a></h3>
<p>제안된 import API는 큰 정수에 효율적입니다. Python 내부(<code>internals</code>)에 직접 접근하는 것과 비교할 때, 제안된 import API는 작은 정수에서 상당한 성능 오버헤드를 가질 수 있습니다.</p>
<p>몇 개의 digit을 가진 작은 정수(예: 1 또는 2 digit)의 경우, 기존 API를 사용할 수 있습니다:</p>
<ul>
<li><code>PyLong_FromUInt64()</code></li>
<li><code>PyLong_FromLong()</code></li>
<li><code>PyLong_FromNativeBytes()</code></li>
</ul>

<h2 id="하위-호환성-backwards-compatibility"><a href="#하위-호환성-backwards-compatibility">하위 호환성 (Backwards Compatibility)</a></h2>
<p>새로운 API만 추가되므로 하위 호환성에 미치는 영향은 없습니다.</p>
<h2 id="기각된-아이디어-rejected-ideas"><a href="#기각된-아이디어-rejected-ideas">기각된 아이디어 (Rejected Ideas)</a></h2>
<h3 id="임의의-레이아웃-지원-support-arbitrary-layout"><a href="#임의의-레이아웃-지원-support-arbitrary-layout">임의의 레이아웃 지원 (Support arbitrary layout)</a></h3>
<p>Python 정수를 import-export하기 위해 임의의 레이아웃을 지원하는 것은 편리할 수 있습니다. 예를 들어, <code>PyLongWriter_Create()</code>에 <code>layout</code> 매개변수를 추가하고 <code>PyLongExport</code> 구조체에 <code>layout</code> 멤버를 추가하는 것이 제안되었습니다.
그러나 구현이 더 복잡하고 실제로 필요하지 않습니다. 엄밀히 필요한 것은 Python의 "네이티브(native)" 레이아웃을 사용하여 import-export하는 API뿐입니다.
나중에 임의의 레이아웃에 대한 사용 사례가 있다면 새로운 API를 추가할 수 있습니다.</p>
<h3 id="pylong_getnativelayout-함수를-추가하지-않음-dont-add-pylong_getnativelayout-function"><a href="#pylong_getnativelayout-함수를-추가하지-않음-dont-add-pylong_getnativelayout-function"><code>PyLong_GetNativeLayout()</code> 함수를 추가하지 않음 (Don't add PyLong_GetNativeLayout() function)</a></h3>
<p>현재 대부분의 정수 import/export에 필요한 정보는 <code>PyLong_GetInfo()</code> (및 <code>sys.int_info</code>)를 통해 이미 사용 가능합니다. 또한, 더 많은 정보(예: digits 순서)를 추가할 수 있으며, 이 인터페이스는 <code>PyLongObject</code>의 미래 발전에 어떤 제약도 가하지 않습니다.
문제는 <code>PyLong_GetInfo()</code>가 편리한 C 구조체가 아닌 Python 객체(named tuple)를 반환한다는 점이며, 이는 사람들이 현재의 세미-프라이빗 매크로(<code>PyLong_SHIFT</code> 및 <code>PyLong_BASE</code> 등)를 선호하게 만들 수 있습니다.</p>
<h3 id="mpz_importexport와-유사한-api-제공-provide-mpz_importexport-like-api-instead"><a href="#mpz_importexport와-유사한-api-제공-provide-mpz_importexport-like-api-instead"><code>mpz_import/export</code>와 유사한 API 제공 (Provide mpz_import/export-like API instead)</a></h3>
<p><code>int</code> 객체에서 데이터를 import/export하는 또 다른 접근 방식은 C 확장이 연속적인 버퍼를 제공하고 CPython이 정수의 절대값을 export(또는 import)하는 것을 기대하는 것입니다.
이 접근 방식의 주요 단점은 CPython 측면에서 훨씬 더 복잡하다는 것입니다(즉, 다른 레이아웃 간의 실제 변환). 예를 들어, <code>CPython</code>에서 <code>PyLong_FromNativeBytes()</code> 및 <code>PyLong_AsNativeBytes()</code> 구현(함께 필요한 API의 제한된 버전을 제공)은 약 500줄의 코드를 차지했습니다(현재 구현은 약 100줄).</p>
<h3 id="export-api에서-value-필드를-삭제-drop-value-field-from-the-export-api"><a href="#export-api에서-value-필드를-삭제-drop-value-field-from-the-export-api">export API에서 <code>value</code> 필드를 삭제 (Drop value field from the export API)</a></h3>
<p>이 제안에 따르면, 하나의 export 유형(digit 배열)만 존재하게 됩니다. 주어진 정수에 대해 이러한 뷰를 사용할 수 없는 경우, export 함수에 의해 에뮬레이션되거나 <code>PyLong_Export()</code>가 오류를 반환할 것입니다. 두 경우 모두, 사용자는 <code>PyLong_AsLongAndOverflow()</code>와 같이 "충분히 작은" 정수(즉, 일부 머신 정수 형식에 맞는 정수)를 얻기 위해 다른 C-API 함수를 사용할 것으로 가정합니다. 이 경우 <code>PyLong_Export()</code>는 비효율적이거나 실패할 것입니다.
이는 API 설계자 관점에서는 단순화처럼 보일 수 있지만, 최종 사용자에게는 덜 편리할 것입니다. 사용자는 Python 개발을 따라야 하고, 작은 정수를 export하기 위한 다양한 변형을 벤치마크하며(위의 경우 <code>PyLong_AsInt64()</code> 대신 선택된 이유가 명확한가요?), 다양한 CPython 버전 또는 다른 Python 구현에 대해 다른 코드 경로를 지원해야 할 수도 있습니다.</p>
