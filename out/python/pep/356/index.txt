<h2>PEP 356 – Python 2.5 릴리스 스케줄</h2>
<p>이 문서는 Python 2.5의 개발 및 릴리스 스케줄을 설명하는 PEP(Python Enhancement Proposal)입니다. 주로 PEP 규모의 항목들을 다루며, 소규모 기능은 첫 번째 베타 릴리스까지 추가될 수 있고, 버그는 최종 릴리스까지 수정될 수 있습니다.</p>
<h3>개요 (Abstract)</h3>
<p>PEP 356은 Python 2.5의 개발 및 릴리스 스케줄을 상세히 기술합니다. 주요 관심사는 PEP 규모의 제안들이며, 작은 기능들은 첫 번째 베타 릴리스까지 포함될 수 있었습니다. 버그는 최종 릴리스 전까지 수정 가능했습니다.</p>
<p>최소 두 번의 알파 릴리스, 두 번의 베타 릴리스, 그리고 한 번의 릴리스 후보(Release Candidate)가 계획되었으며, 최종 릴리스는 2006년 9월 12일로 예정되었습니다.</p>
<h3>릴리스 관리자 (Release Manager)</h3>
<p>Anthony Baxter가 릴리스 관리자(Release Manager)를 맡았습니다. Martin von Loewis는 Windows 설치 프로그램을, Ronald Oussoren은 Mac 설치 프로그램을, Fred Drake는 문서 패키지를, Sean Reifschneider는 RPM을 빌드했습니다.</p>
<h3>릴리스 스케줄 (Release Schedule)</h3>
<p>Python 2.5의 릴리스 스케줄은 다음과 같이 진행되었습니다 (모두 완료됨):</p>
<ul>
<li><strong>alpha 1:</strong> 2006년 4월 5일</li>
<li><strong>alpha 2:</strong> 2006년 4월 27일</li>
<li><strong>beta 1:</strong> 2006년 6월 20일</li>
<li><strong>beta 2:</strong> 2006년 7월 11일</li>
<li><strong>beta 3:</strong> 2006년 8월 3일</li>
<li><strong>rc 1:</strong> 2006년 8월 17일</li>
<li><strong>rc 2:</strong> 2006년 9월 12일</li>
<li><strong>final:</strong> 2006년 9월 19일</li>
</ul>
<h3>Python 2.5에 포함된 주요 기능들 (Completed features for 2.5)</h3>
<p>Python 2.5 릴리스에는 다음과 같은 중요한 PEP 및 기능들이 포함되었습니다:</p>
<h4>PEP 관련 기능들</h4>
<ul>
<li><strong>PEP 308: Conditional Expressions (조건부 표현식)</strong>: <code>A if condition else B</code>와 같은 삼항 연산자 도입.</li>
<li><strong>PEP 309: Partial Function Application (부분 함수 적용)</strong>: <code>functools.partial</code>을 통한 함수 인자 부분 적용.</li>
<li><strong>PEP 314: Metadata for Python Software Packages v1.1</strong>: Python 소프트웨어 패키지 메타데이터 표준 업데이트.</li>
<li><strong>PEP 328: Absolute/Relative Imports (절대/상대 임포트)</strong>: 모듈 임포트 방식 개선.</li>
<li><strong>PEP 338: Executing Modules as Scripts (모듈을 스크립트로 실행)</strong>: <code>-m</code> 옵션을 통한 모듈 실행 기능.</li>
<li><strong>PEP 341: Unified try-except/try-finally to try-except-finally</strong>: <code>try-except-finally</code> 문의 통합.</li>
<li><strong>PEP 342: Coroutines via Enhanced Generators (강화된 제너레이터를 통한 코루틴)</strong>: <code>yield from</code> (Python 3.3에 도입)의 전신 격인 <code>generator.send()</code> 등 제너레이터 메서드 도입으로 코루틴 구현의 기반을 마련.</li>
<li><strong>PEP 343: The “with” Statement (<code>with</code> 문)</strong>: <code>with</code> 문 도입으로 컨텍스트 관리 프로토콜(<code>__enter__</code>, <code>__exit__</code>)을 사용하는 리소스 관리를 간편하게 함 (예: 파일, 락).</li>
<li><strong>PEP 352: Required Superclass for Exceptions (예외를 위한 필수 슈퍼클래스)</strong>: 모든 사용자 정의 예외가 <code>BaseException</code> (또는 그 서브클래스)를 상속하도록 강제.</li>
<li><strong>PEP 353: Using ssize_t as the index type (인덱스 타입으로 ssize_t 사용)</strong>: C API에서 인덱스 타입으로 <code>ssize_t</code>를 사용하여 더 큰 시퀀스 지원.</li>
<li><strong>PEP 357: Allowing Any Object to be Used for Slicing (어떤 객체든 슬라이싱에 사용 가능)</strong>: 슬라이싱 작업에 더 다양한 객체를 사용할 수 있도록 허용.</li>
</ul>
<h4>기타 주목할 만한 기능들</h4>
<ul>
<li>기본 인코딩이 ASCII로 변경되었습니다.</li>
<li>AST(Abstract Syntax Tree) 기반 컴파일러가 도입되었고, 새로운 <code>_ast</code> 모듈을 통해 Python에서 C AST에 접근할 수 있게 되었습니다.</li>
<li><code>any()</code> 및 <code>all()</code> 빌트인(Built-in) 진리 함수가 추가되었습니다.</li>
<li><strong>새로운 표준 라이브러리 모듈:</strong>
<ul>
<li><code>cProfile</code>: 최소한의 오버헤드로 장시간 실행되는 애플리케이션 프로파일링에 적합.</li>
<li><code>ctypes</code>: Windows 설치 프로그램의 선택적 구성 요소. 외부 C 라이브러리 호출을 지원.</li>
<li><code>ElementTree</code> 및 <code>cElementTree</code>: XML 파싱 및 생성을 위한 라이브러리.</li>
<li><code>hashlib</code>: SHA-224, -256, -384, -512 지원을 추가 (기존 <code>md5</code> 및 <code>sha</code> 모듈 대체).</li>
<li><code>msilib</code>: MSI 파일 생성 및 <code>distutils</code>의 <code>bdist_msi</code>를 위함.</li>
<li><code>pysqlite</code>: SQLite 데이터베이스 바인딩.</li>
<li><code>uuid</code>: UUID(Universally Unique Identifier) 생성.</li>
<li><code>wsgiref</code>: WSGI(Web Server Gateway Interface) 참조 구현.</li>
</ul>
</li>
<li>쉐도우 패스워드(shadow passwords) 읽기 지원이 추가되었습니다.</li>
<li>Unicode 4.1 UCD 지원이 추가되었습니다.</li>
<li><code>warnings</code>, <code>linecache</code>, <code>inspect</code>, <code>traceback</code>, <code>site</code>, <code>doctest</code> 모듈에 PEP 302 <code>zipfile</code> / <code>__loader__</code> 지원이 추가되었습니다.</li>
<li><code>pybench</code> Python 벤치마크 스위트가 추가되었습니다.</li>
<li>메일박스(mailboxes)에 대한 쓰기 지원이 추가되었습니다.</li>
<li>"fat" Mac 바이너리(Intel 및 PPC) 빌드 지원이 추가되었습니다.</li>
<li><code>functools</code>에 래퍼(wrapper) 함수 작성을 돕는 새로운 유틸리티가 추가되었습니다.</li>
<li><code>pyexpat</code>가 expat 2.0을 사용하도록 업그레이드되었습니다.</li>
<li>Python 코어가 <code>g++</code>로 깨끗하게 컴파일되도록 개선되었습니다.</li>
</ul>
<h3>2.6으로 연기된 기능들 (Deferred until 2.6)</h3>
<ul>
<li><code>distutils</code> 패키지의 <code>bdist_deb</code>.</li>
<li><code>distutils</code> 패키지의 <code>bdist_egg</code>.</li>
<li>순수 Python <code>pgen</code> 모듈.</li>
<li><code>fpectl</code> 모듈 제거 (고려).</li>
<li><code>Modules/</code> 내의 모든 것이 <code>g++</code>로 깔끔하게 빌드되도록 변경하는 작업.</li>
</ul>
<h3>오픈 이슈 (Open issues)</h3>
<p>릴리스 전에 해결되어야 할 버그는 없었으나, 2.5.1 또는 그 이후 버전으로 연기된 몇몇 버그와 문서화 및 테스트 관련 이슈들이 있었습니다.</p>
<ul>
<li><code>socket</code> 모듈의 스레드 안전성 문제 (https://bugs.python.org/issue1544279).</li>
<li>Windows에서 <code>tools</code> 및 <code>demo</code> 누락 문제 (https://bugs.python.org/issue1541420).</li>
<li>중첩된 <code>try/finally</code>에서 <code>continue</code> 사용 시 크래시 (https://bugs.python.org/issue1542451).</li>
<li><code>gettext.py</code> 버그 (https://bugs.python.org/issue1475523).</li>
<li><code>%-formatting</code> 및 <code>dicts</code> 관련 문제 (https://bugs.python.org/issue1467929).</li>
<li><code>unicode()</code>가 <code>LookupError</code>를 발생시키지 않는 문제 (https://bugs.python.org/issue1446043).</li>
</ul>
<p>또한, <code>pkgutil</code>, <code>runpy</code>, <code>pydoc</code>에 대한 PEP 302 변경 사항이 문서화되어야 했고, <code>test_zipfile64</code> 테스트의 긴 실행 시간 및 디스크 공간 문제, 그리고 문서화되지 않은 C 모듈의 제거 여부 등이 논의되었습니다.</p>
