<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0299/" target="_blank" rel="noopener noreferrer">PEP 299 - Special <strong>main</strong>() function in modules</a></p>

<h1 id="pep-299-모듈-내-특별한-__main__-함수"><a href="#pep-299-모듈-내-특별한-__main__-함수">PEP 299 – 모듈 내 특별한 <code>__main__()</code> 함수</a></h1>
<ul>
<li><strong>작성자 (Author):</strong> Jeff Epler <jepler at unpythonic.net></li>
<li><strong>상태 (Status):</strong> 반려됨 (Rejected)</li>
<li><strong>유형 (Type):</strong> 표준 트랙 (Standards Track)</li>
<li><strong>작성일 (Created):</strong> 2002년 8월 12일</li>
<li><strong>Python 버전 (Python-Version):</strong> 2.3</li>
<li><strong>게시 이력 (Post-History):</strong> 2006년 3월 29일</li>
</ul>
<h2 id="요약-abstract"><a href="#요약-abstract">요약 (Abstract)</a></h2>
<p>많은 Python 모듈은 독립 실행형 스크립트(standalone scripts)로도 호출될 수 있도록 의도되었습니다. 이 PEP는 <code>__main__()</code>이라는 특별한 함수가 이 목적을 수행해야 한다고 제안합니다.</p>
<h2 id="도입-배경-motivation"><a href="#도입-배경-motivation">도입 배경 (Motivation)</a></h2>
<p>모듈을 독립 실행형 스크립트로 호출하기 위한 하나의 간단하고 보편적인 관용구(idiom)가 있어야 합니다.</p>
<p>현재의 "반쯤 표준적인(semi-standard)" 관용구는 다음과 같습니다:</p>
<pre><code class="language-python">if __name__ == '__main__':
    perform "standalone" functionality
</code></pre>
<p>이 방식은 C나 C++와 같은 언어의 프로그래머들에게는 불분명합니다. 또한, 모듈이 임포트(import)될 때 독립 실행형 함수를 호출하는 것을 허용하지 않습니다.</p>
<p>때때로 다음과 같은 변형된 형태가 사용되기도 합니다:</p>
<pre><code class="language-python">if __name__ == '__main__':
    main_function()
</code></pre>
<p>하지만 이 함수에 대한 표준 이름이 존재하지 않으며, 인자(arguments)가 <code>sys.argv</code>에서 가져와지기 때문에 다른 모든 모듈이 보는 인자 리스트를 변경하지 않고는 특정 인자를 전달할 수 없습니다. (예를 들어, 두 개의 스레드가 서로 다른 인자 리스트로 다른 모듈의 독립 실행형 기능을 호출하려는 멀티스레드(threaded) Python 프로그램을 상상해 보세요.)</p>
<h2 id="제안-proposal"><a href="#제안-proposal">제안 (Proposal)</a></h2>
<p>'메인 함수(main function)'의 표준 이름은 <code>__main__</code>이 되어야 합니다. 모듈이 다음과 같이 명령줄(command line)에서 호출될 때:</p>
<pre><code class="language-bash">python mymodule.py
</code></pre>
<p>그러면 모듈은 해당 모듈의 끝에 다음 줄이 존재하는 것처럼 동작합니다 (단, <code>__sys</code> 속성은 스크립트의 다른 곳에서 사용되거나 존재한다고 가정할 수 없습니다):</p>
<pre><code class="language-python">if globals().has_key("__main__"):
    import sys as __sys
    __sys.exit(__main__(__sys.argv))
</code></pre>
<p>다른 모듈은 다음과 같이 실행할 수 있습니다:</p>
<pre><code class="language-python">import mymodule
mymodule.__main__(['mymodule', ...])
</code></pre>
<p><code>__main__</code>의 사용을 제한할 수 있는 스레드 안전성(thread-safety) 문제나 다른 문제들(예: 상호 배타적인 GUI 모듈 사용, 하드웨어 장치와 같이 공유할 수 없는 자원, <code>sys.stdin</code> / <code>stdout</code> 재할당 등)을 문서화하는 것은 <code>mymodule</code>의 책임입니다.</p>
<h2 id="구현-implementation"><a href="#구현-implementation">구현 (Implementation)</a></h2>
<p><code>modules/main.c</code> 파일의 385행 근처 블록( <code>PyRun_AnyFileExFlags</code> 호출 후)이 변경되어 위의 코드(또는 그에 상응하는 C 코드)가 실행되도록 할 것입니다.</p>
<h2 id="해결되지-않은-문제들-open-issues"><a href="#해결되지-않은-문제들-open-issues">해결되지 않은 문제들 (Open Issues)</a></h2>
<ul>
<li><code>__main__</code>의 반환 값은 종료 값(exit value)으로 처리되어야 하는가?
<ul>
<li><strong>예.</strong> 많은 <code>__main__</code> 함수는 자연스럽게 <code>None</code>을 반환하며, <code>sys.exit</code>는 이를 "성공" 반환 코드로 변환합니다. 숫자 결과를 반환하는 경우, 이는 <code>sys.exit()</code>의 인자나 C의 <code>main()</code> 함수의 반환 값과 동일하게 동작합니다.</li>
</ul>
</li>
<li><code>__main__</code>으로 전달되는 인자 리스트에 <code>argv[0]</code>이 포함되어야 하는가, 아니면 "실제" 인자인 <code>argv[1:]</code>만 포함되어야 하는가?
<ul>
<li><code>sys.argv</code>와의 대칭성 및 새로운 표준 관용구로의 쉬운 전환을 위해 <code>argv[0]</code>이 포함됩니다.</li>
</ul>
</li>
</ul>
<h2 id="반려-rejection"><a href="#반려-rejection">반려 (Rejection)</a></h2>
<p><code>python-dev</code>에서의 짧은 논의에서 두 가지 주요 하위 호환성(backwards compatibility) 문제가 제기되었으며, 귀도(Guido van Rossum)는 어쨌든 이 아이디어를 좋아하지 않는다고 말했습니다. 그는 "문서, 사용자 습관 등의 변경 가치가 없으며, 특별히 고장난 것도 없다"고 언급했습니다.</p>
