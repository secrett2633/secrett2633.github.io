<p><strong>상태:</strong> Rejected | <strong>유형:</strong> Informational | <strong>작성일:</strong> 03-Jul-2015</p>

<h1>PEP 496 – 환경 마커 (Environment Markers) 번역 및 해설</h1>
<h2>PEP 상태</h2>
<p>이 PEP가 처음 초안으로 작성된 후, 환경 마커를 포함한 의존성 선언 구문을 완전히 명세화하기 위해 PEP 508이 개발 및 제출되었습니다. 그 결과, 이 PEP는 더 포괄적인 PEP 508에 찬성하여 <code>거부(Rejected)</code>되었습니다.</p>
<h2>개요</h2>
<p><code>환경 마커(Environment Marker)</code>는 현재 실행 환경에 대한 조건을 설명합니다. 이는 특정 의존성이 특정 환경에서만 필요할 때, 그리고 Python 런타임 가용성을 넘어 추가적인 제약 조건이 있는 배포판의 지원 플랫폼을 나타낼 때 사용됩니다.</p>
<p>환경 마커는 PEP 345에서 처음 명세화되었습니다. PEP 426 (PEP 345를 대체할 예정이었던)은 마커에 대한 확장을 제안했습니다. 2.7.10 버전이 출시되었을 때, 단순한 어휘적 비교에 의존했기 때문에 이러한 확장조차 불충분해졌고, 이에 따라 이 PEP가 탄생하게 되었습니다.</p>
<h2>배경 (Rationale)</h2>
<p>많은 Python 패키지는 이식성을 염두에 두고 작성됩니다.</p>
<p>많은 패키지의 경우, 이는 광범위한 Python 릴리스를 지원하는 것을 목표로 합니다. 예를 들어, <code>argparse</code>와 같이 외부 라이브러리로 시작했지만 나중에 코어에 통합된 라이브러리에 의존하는 경우, 사용 중인 Python 버전에 따라 필요한 패키지 세트가 달라지므로 단일 요구 사항 세트를 지정하기 어렵습니다.</p>
<p>다른 패키지의 경우, 이식성을 위해 여러 운영 체제를 지원해야 합니다. 그러나 운영 체제 간의 상당한 차이로 인해 특정 의존성이 특정 플랫폼에서만 필요할 수 있습니다 (예: Windows에서만 <code>pywin32</code>에 의존하는 경우).</p>
<p>환경 마커는 개발자가 특정 환경에 특화된 요구 사항을 나열할 수 있도록 하여 요구 사항 목록에 더 많은 유연성을 제공하고자 합니다.</p>
<h2>예시 (Examples)</h2>
<p>다음은 <code>requirements.txt</code> 내에서 이러한 마커를 사용하는 몇 가지 예시입니다.</p>
<pre><code>pywin32 >=1.0 ; sys_platform == 'win32'
unittest2 >=2.0,&#x3C;3.0 ; python_version == '2.4' or python_version == '2.5'
backports.ssl_match_hostname >= 3.4 ; python_version &#x3C; '2.7.9' or (python_version >= '3.0' and python_version &#x3C; '3.4')
</code></pre>
<p>다음은 Windows를 사용할 때 런타임과 빌드 타임 모두에서 PyWin32를 요구하는 배포판의 <code>setup.py</code>에 포함된 조건부 메타데이터의 예시입니다.</p>
<pre><code class="language-python">setup(
    install_requires=["pywin32 > 1.0 : sys.platform == 'win32'"],
    setup_requires=["pywin32 > 1.0 : sys.platform == 'win32'"]
)
</code></pre>
<h2>마이크로 언어 (Micro-language)</h2>
<p>이 마이크로 언어의 동작 방식은 다음과 같습니다.</p>
<ul>
<li>문자열을 <code>==</code> 및 <code>in</code> 연산자(및 그 반대)로 비교합니다.</li>
<li>버전 번호를 문자열에 지원되는 연산자 외에 <code>&#x3C;</code>, <code>&#x3C;=</code>, <code>>=</code>, <code>></code> 연산자로 비교합니다.</li>
</ul>
<p>일반적인 부울 연산자 <code>and</code> 및 <code>or</code>를 사용하여 표현식을 결합할 수 있으며, 그룹화를 위해 괄호가 지원됩니다.</p>
<p>의사 문법(pseudo-grammar)은 다음과 같습니다.</p>
<pre><code>MARKER: EXPR [(and|or) EXPR]*
EXPR: ("(" MARKER ")") | (STREXPR|VEREXPR)
STREXPR: STRING [STRCMPOP STREXPR]
STRCMPOP: ==|!=|in|not in
VEREXPR: VERSION [VERCMPOP VEREXPR]
VERCMPOP: (==|!=|&#x3C;|>|&#x3C;=|>=)
</code></pre>
<p><code>SUBEXPR</code>은 Python 문자열(예: <code>'win32'</code>)이거나 아래 나열된 <code>Strings</code> 마커 변수 중 하나입니다.</p>
<p><code>VEREXPR</code>은 PEP 440 버전 식별자이거나 아래 나열된 <code>Version number</code> 마커 변수 중 하나입니다. 버전 번호 간의 비교는 PEP 440 의미 체계를 사용하여 수행됩니다.</p>
<h3>문자열 (Strings)</h3>
<ul>
<li><code>os_name</code>: <code>os.name</code></li>
<li><code>sys_platform</code>: <code>sys.platform</code></li>
<li><code>platform_release</code>: <code>platform.release()</code></li>
<li><code>implementation_name</code>: <code>sys.implementation.name</code></li>
<li><code>platform_machine</code>: <code>platform.machine()</code></li>
<li><code>platform_python_implementation</code>: <code>platform.python_implementation()</code></li>
</ul>
<p>특정 문자열 값(예: Python 3.3 이전 버전의 <code>sys.implementation.name</code>)을 사용할 수 없는 경우, 해당 마커 변수는 빈 문자열과 동일하게 간주되어야 합니다.</p>
<p>특정 버전 번호 값(예: Python 3.3 이전 버전의 <code>sys.implementation.version</code>)을 사용할 수 없는 경우, 해당 마커 변수는 0과 동일하게 간주되어야 합니다.</p>
<h3>버전 번호 (Version numbers)</h3>
<ul>
<li><code>python_version</code>: <code>platform.python_version()[:3]</code></li>
<li><code>python_full_version</code>: 아래 정의 참조</li>
<li><code>platform_version</code>: <code>platform.version()</code></li>
<li><code>implementation_version</code>: 아래 정의 참조</li>
</ul>
<p><code>python_full_version</code> 및 <code>implementation_version</code> 마커 변수는 다음 알고리즘에 따라 각각 <code>sys.version_info</code> 및 <code>sys.implementation.version</code>에서 파생됩니다.</p>
<pre><code class="language-python">def format_full_version(info):
    version = '{0.major}.{0.minor}.{0.micro}'.format(info)
    kind = info.releaselevel
    if kind != 'final':
        version += kind[0] + str(info.serial)
    return version

python_full_version = format_full_version(sys.version_info)
implementation_version = format_full_version(sys.implementation.version)
</code></pre>
<p><code>python_full_version</code>은 일반적으로 <code>sys.version.split()[0]</code>에 해당합니다.</p>
