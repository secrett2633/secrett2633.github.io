<p><strong>상태:</strong> Final | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 08-Mar-2002</p>
</blockquote>
<h2>PEP 285 – <code>bool</code> 타입 추가 제안</h2>
<h3>개요</h3>
<p>PEP 285는 Python에 새로운 내장(built-in) 타입인 <code>bool</code>을 도입하고, 두 개의 상수 <code>False</code>와 <code>True</code>를 추가할 것을 제안합니다. 이 <code>bool</code> 타입은 내부적으로 <code>int</code> 타입의 서브타입(subtype)으로 구현되며, <code>False</code>는 <code>0</code>처럼, <code>True</code>는 <code>1</code>처럼 동작합니다. 그러나 <code>repr()</code> 및 <code>str()</code> 함수 호출 시에는 <code>0</code>이나 <code>1</code> 대신 <code>"False"</code>나 <code>"True"</code>를 반환합니다.</p>
<p>이 제안이 수락됨에 따라, 개념적으로 불리언(Boolean) 결과를 반환하는 모든 내장 연산(예: 비교 연산자, <code>not</code> 연산자, <code>isinstance()</code>와 같은 술어(predicate) 함수)은 더 이상 <code>0</code>이나 <code>1</code> 대신 <code>False</code> 또는 <code>True</code>를 반환하도록 변경되었습니다.</p>
<h3>도입 배경 (Rationale)</h3>
<p>대부분의 프로그래밍 언어는 결국 불리언 타입을 가지게 되며, 심지어 C99 표준에도 불리언 타입이 포함되어 있습니다. 많은 프로그래머들이 불리언 타입의 필요성을 느끼고 있었으며, 기존 Python 문서에서도 불리언 타입의 부재에 대한 설명을 포함하곤 했습니다.</p>
<p>과거에는 많은 개발자가 <code>False=0</code>과 <code>True=1</code>과 같은 상수를 직접 정의하여 사용했지만, 이 방식은 일관성이 부족하다는 문제가 있었습니다. 예를 들어, <code>FALSE</code>, <code>false</code>, <code>False</code>, <code>F</code>, <code>f</code> 중 어떤 표기법을 사용해야 할지, 또는 <code>false</code>가 <code>0</code>, <code>None</code>, 혹은 다른 타입의 진릿값(truth value)이어야 하는지 등 혼란이 있었습니다. 표준 <code>bool</code> 타입을 언어에 추가함으로써 이러한 문제를 해결할 수 있게 되었습니다.</p>
<p>또한, 일부 외부 라이브러리(데이터베이스, RPC 패키지 등)는 불리언 값과 정수(integral) 값을 구별해야 할 필요가 있었습니다. 자바(Jython)와 같은 환경에서는 <code>int</code>와 <code>boolean</code> 인수에 대해 오버로드(overload)된 메서드가 존재하여 <code>bool</code> 타입이 이러한 상황에서 적절한 메서드를 선택하는 데 사용될 수 있습니다.</p>
<p>값을 불리언으로 강제 변환하거나 정규화할 때 <code>bool(x)</code>는 <code>not not x</code>보다 훨씬 명확하며, <code>if x: return 1 else: return 0</code>과 같은 코드보다 간결합니다. 교육적인 측면에서도 <code>a > b</code>와 같은 비교 연산의 결과가 <code>1</code> 대신 <code>True</code>로 표시되는 것이 초보 학습자에게 더 직관적이고 이해하기 쉽습니다.</p>
<h3>명세 (Specification)</h3>
<p>새로운 <code>bool</code> 타입의 주요 특징은 다음과 같습니다.</p>
<ul>
<li><code>bool</code>은 <code>int</code>를 상속합니다.
<pre><code class="language-python">class bool(int):
    def __new__(cls, val=0):
        if val:
            return True
        else:
            return False
    def __repr__(self):
        if self:
            return "True"
        else:
            return "False"
    __str__ = __repr__
    # ... (and, or, xor 연산자 오버라이드)
</code></pre>
</li>
<li><code>False</code>와 <code>True</code>는 <code>None</code>과 같은 싱글턴(singleton)입니다. 즉, 단 하나의 인스턴스만 존재합니다.</li>
<li><code>pickle</code> 및 <code>marshal</code> 모듈을 통한 직렬화/역직렬화(pickling and marshalling) 과정에서 <code>True</code>와 <code>False</code>는 올바르게 round-trip 됩니다.</li>
<li>개념적으로 불리언 결과를 반환하는 모든 내장 연산은 <code>0</code> 또는 <code>1</code> 대신 <code>False</code> 또는 <code>True</code>를 반환하도록 변경됩니다. 이는 비교 연산자(<code>==</code>, <code>&#x3C;</code>, <code>is</code> 등), 단항 연산자 <code>not</code>, <code>callable()</code>, <code>hasattr()</code>, <code>isinstance()</code>, <code>issubclass()</code>와 같은 내장 함수, 그리고 <code>str</code>, <code>unicode</code>, <code>dict</code>, <code>file</code> 객체의 여러 메서드에 영향을 미칩니다.</li>
<li><code>bool</code>이 <code>int</code>를 상속하므로, <code>True + 1</code>은 유효하며 결과는 <code>2</code>입니다. 이는 하위 호환성(backwards compatibility)을 위해 중요합니다.</li>
</ul>
<h3>C API</h3>
<p><code>bool</code> 타입의 C API는 "boolobject.h" 헤더 파일에 정의되어 있으며, "Python.h"에 포함되어 있습니다.</p>
<ul>
<li><code>Py_False</code>와 <code>Py_True</code>는 고유한 <code>False</code>와 <code>True</code> 불리언 객체를 참조합니다.</li>
<li>새로운 API인 <code>PyObject *PyBool_FromLong(long)</code>는 C의 <code>long int</code> 인수를 받아 <code>Py_False</code> 또는 <code>Py_True</code>에 대한 새 참조를 반환합니다.</li>
<li>객체가 <code>bool</code>인지 확인하려면 매크로 <code>PyBool_Check()</code>를 사용할 수 있습니다.</li>
</ul>
<h3>명확화 (Clarification)</h3>
<p>이 PEP는 거의 모든 객체 타입이 진릿값으로 사용될 수 있다는 사실을 변경하지 않습니다. 예를 들어, <code>if</code> 문에서 빈 리스트는 <code>False</code>로 간주되고 비어 있지 않은 리스트는 <code>True</code>로 간주되는 기존의 동작은 변하지 않으며, 이를 변경할 계획도 없습니다.</p>
<p>변경되는 유일한 점은 진릿값을 명시적으로 반환하거나 할당할 때 선호되는 값입니다. 이전에는 <code>0</code>과 <code>1</code>이 선호되었지만, 이 PEP는 선호되는 값을 <code>False</code>와 <code>True</code>로 변경하고, 내장 연산이 이러한 선호 값을 반환하도록 수정합니다.</p>
<h3>호환성 (Compatibility)</h3>
<p>하위 호환성을 위해 <code>bool</code> 타입은 일부 개발자들이 원했을 만한 엄격한 불리언 특성을 모두 가지지는 않습니다. 예를 들어, 하나 또는 두 개의 <code>bool</code> 인수를 사용하는 산술 연산이 허용되며, <code>False</code>는 <code>0</code>으로, <code>True</code>는 <code>1</code>로 취급됩니다. 또한, <code>bool</code>은 시퀀스(sequence) 인덱스로 사용될 수 있습니다.</p>
<p><code>and</code> 및 <code>or</code> 연산자는 결과를 결정하는 첫 번째 인수를 반환하도록 정의되어 있으며, 이는 변경되지 않습니다. 따라서 이들 연산자는 항상 불리언 값을 강제하지 않습니다. 물론, 두 인수가 모두 <code>bool</code>이면 결과도 항상 <code>bool</code>입니다. 결과를 <code>bool</code>로 강제하려면 <code>bool(x and y)</code>와 같이 작성할 수 있습니다.</p>
<h3>해결된 문제 (Resolved Issues)</h3>
<ul>
<li><code>repr()</code> 또는 <code>str()</code> 함수의 결과가 <code>int</code>와 다르기 때문에 일부 코드(예: <code>doctest</code> 기반 유닛 테스트)가 실패할 수 있지만, 쉽게 수정할 수 있는 적은 양의 코드에만 영향을 미칠 것으로 예상됩니다.</li>
<li>다른 언어(C99, C++, Java)에서는 상수를 소문자 <code>false</code>, <code>true</code>로 사용하지만, Python은 <code>None</code>, <code>Ellipsis</code>, <code>NotImplemented</code>와 같은 기존 내장 상수의 대문자 규칙을 따르기로 결정했습니다.</li>
<li>초보자들이 <code>if x == True:</code>와 같이 작성하는 경향이 있지만, 올바른 형식은 <code>if x:</code>입니다. <code>x == True</code>가 <code>True</code>로 간주되는 모든 <code>x</code>에 대해 참이 되도록 하는 것은 <code>6 == True == 7</code>과 같은 논리적 오류를 발생시킬 수 있으므로 허용되지 않습니다. 개발자 교육을 통해 올바른 사용법을 알리는 것이 중요하다고 제안되었습니다.</li>
<li><code>if bool(x):</code>와 같이 명시적으로 <code>bool()</code>을 사용하는 것은 대개 필요하지 않으며 가독성을 저해할 수 있습니다. 그러나 임의의 객체 <code>x</code>에 대한 참조를 유지하는 것이 번거롭거나 정규화가 필요한 경우 <code>b = bool(x)</code>와 같이 사용하는 것은 유용할 수 있습니다. 값을 정수로 계속 사용하겠다는 의도를 명확히 하기 위해 <code>i = int(bool(x))</code>와 같이 사용할 수도 있습니다.</li>
</ul>
