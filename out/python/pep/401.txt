<h1 id="pep-401-bdfl-은퇴"><a href="#pep-401-bdfl-은퇴">PEP 401 – BDFL 은퇴</a></h1>
<ul>
<li><strong>작성자:</strong> Barry Warsaw, Brett Cannon</li>
<li><strong>상태:</strong> 만우절! (April Fool!)</li>
<li><strong>유형:</strong> Process (프로세스)</li>
<li><strong>생성일:</strong> 2009년 4월 1일</li>
<li><strong>후속 기록:</strong> 2009년 4월 1일</li>
</ul>
<h2 id="개요-abstract"><a href="#개요-abstract">개요 (Abstract)</a></h2>
<p>20년 동안 Python 개발을 이끌어온 BDFL이 즉시 은퇴를 공식적으로 발표합니다. 만장일치 투표를 통해 그의 후임자가 지명되었습니다.</p>
<h2 id="배경-rationale"><a href="#배경-rationale">배경 (Rationale)</a></h2>
<p>Guido는 1989년에 Python의 초기 구현을 작성했으며, 거의 20년 동안 커뮤니티를 이끌어온 후 '평생의 자비로운 독재자(Benevolent Dictator For Life, BDFL)' 자리에서 물러나기로 결정했습니다. 그의 공식 직함은 이제 '언어로부터 무기한 휴가 중인 자비로운 독재자 명예회장 (Benevolent Dictator Emeritus Vacationing Indefinitely from the Language, BDEVIL)'이 됩니다. Guido는 에베레스트 등반이라는 평생의 꿈을 위해 훈련하기 위해 Python을 새로운 리더와 활기찬 커뮤니티의 좋은 손에 맡깁니다.</p>
<p>2009년 Python Conference (PyCon 2009)에서 Python Steering Union (확실히 존재하지 않는 Python Secret Underground와 혼동하지 마세요)의 만장일치 투표 끝에 Guido의 후임자로 Barry Warsaw, 애칭으로 "Uncle Barry"가 선정되었습니다. Uncle Barry의 공식 직함은 '평생의 친근한 언어 삼촌 (Friendly Language Uncle For Life, FLUFL)'입니다.</p>
<h2 id="flufl의-공식적인-결정-official-acts-of-the-flufl"><a href="#flufl의-공식적인-결정-official-acts-of-the-flufl">FLUFL의 공식적인 결정 (Official Acts of the FLUFL)</a></h2>
<p>FLUFL Uncle Barry는 그의 전임자(이름을 기억하지 못함)와 동일하게 책임감 있고 개방적인 방식으로 커뮤니티를 이끌겠다는 의지를 보여주기 위해 다음 결정을 시행합니다.</p>
<ul>
<li><strong>DVCS 변경:</strong> Hg (Mercurial)를 분산 버전 관리 시스템 (DVCS)으로 선택한 것은 BDEVIL의 정신 이상 발병의 명확한 증거임을 인식하고, 이 결정을 취소하여 유일한 진정한 선택인 Bzr (Bazaar)로 전환합니다.</li>
<li><strong><code>!=</code> 연산자 변경:</strong> Python 3.0의 <code>!=</code> (불평등) 연산자는 끔찍하고 손가락 통증을 유발하는 실수였음을 인식하고, FLUFL은 <code>기호</code>를 유일한 표기로 복원합니다. 이 변경 사항은 Python 3.1에 구현 및 출시될 만큼 중요합니다. 이 기능으로의 전환을 돕기 위해 새로운 future statement인 <code>from __future__ import barry_as_FLUFL</code>이 추가되었습니다.</li>
<li><strong><code>print()</code> 함수 변경:</strong> Python 3.0의 <code>print()</code> 함수는 끔찍하고 고통을 유발하는 실수였음을 인식하고, FLUFL은 <code>print</code> 문(statement)을 복원합니다. 이 변경 사항은 Python 3.0.2에 구현 및 출시될 만큼 중요합니다.</li>
<li><strong>Python 3.x 개발 종료:</strong> Python 3.0의 실망스러운 채택 곡선이 완전한 실패를 알린다고 인식하여, Python 3.1 및 이후의 Python 3.x 버전의 모든 작업은 이로써 종료됩니다. Python 3.0의 모든 기능은 Python 2.7로 백포트될 것이며, Python 2.7이 공식적인 유일한 다음 릴리스가 될 것입니다. 개발자들의 편의를 위해 Python 3.0의 string 및 bytes 타입은 Python 2.6.2로 백포트됩니다.</li>
<li><strong>CPython 개발 종료 및 Parrot VM으로 전환:</strong> C가 30세 미만 프로그래머들로부터 거의 보편적으로 거부당하는 20세기 언어임을 인식하여, CPython 구현은 Python 2.6.2 및 3.0.2 릴리스와 함께 종료됩니다. 이후 Python의 참조 구현은 Parrot 가상 머신 을 목표로 합니다. 다른 Python 구현 (예: Jython, IronPython, PyPy)은 공식적으로 권장되지 않지만 허용됩니다.</li>
<li><strong>Python Software Foundation 해체:</strong> Python Software Foundation (PSF)이 그 임무를 훌륭하게 완수했음을 인식하여, 이로써 해체됩니다. Python Steering Union (확실히 존재하지 않는 Python Secret Underground와 혼동하지 마세요)이 이제 모든 Python의 지적 재산에 대한 유일한 관리자가 됩니다. 모든 PSF 자금은 이로써 PSU (그 PSU 말고, 다른 PSU)로 이전됩니다.</li>
</ul>
