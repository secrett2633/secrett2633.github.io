<p>PEP 559 – Built-in <code>noop()</code> | peps.python.org 문서를 번역하고 정리합니다.</p>
<h2 id="pep-559-built-in-noop"><a href="#pep-559-built-in-noop">PEP 559 – Built-in <code>noop()</code></a></h2>
<ul>
<li><strong>작성자:</strong> Barry Warsaw <barry at python.org></li>
<li><strong>상태:</strong> Rejected (거부됨)</li>
<li><strong>유형:</strong> Standards Track</li>
<li><strong>생성일:</strong> 2017년 9월 8일</li>
<li><strong>Python 버전:</strong> 3.7</li>
<li><strong>사후 이력:</strong> 2017년 9월 9일</li>
<li><strong>해결:</strong> Python-Dev message</li>
</ul>
<h3 id="초록-abstract"><a href="#초록-abstract">초록 (Abstract)</a></h3>
<p>이 PEP는 아무것도 하지 않고 <code>None</code>을 반환하는 새로운 내장 함수 <code>noop()</code>를 추가할 것을 제안합니다.</p>
<h3 id="배경-rationale"><a href="#배경-rationale">배경 (Rationale)</a></h3>
<p>Python에서 아무것도 하지 않는(no-op) 함수를 구현하는 것은 아주 쉽습니다. 너무 쉽기 때문에 많은 사람이 반복적으로 여러 번 구현하곤 합니다. 따라서 아무것도 하지 않는 공통의 내장 함수가 있다면 많은 경우에 유용할 것입니다.</p>
<p>하나의 사용 사례는 PEP 553과 관련이 있습니다. <code>breakpoint</code> 환경 변수를 다음과 같이 설정하여 <code>breakpoint</code> 기능을 효과적으로 비활성화할 수 있습니다.</p>
<pre><code class="language-bash">$ setenv PYTHONBREAKPOINT=noop
</code></pre>
<h3 id="구현-implementation"><a href="#구현-implementation">구현 (Implementation)</a></h3>
<p><code>noop()</code> 함수의 Python 동등한 구현은 정확히 다음과 같습니다.</p>
<pre><code class="language-python">def noop(*args, **kws):
    return None
</code></pre>
<p>C 언어로 구현된 내장 함수의 구현은 풀 리퀘스트를 통해 확인할 수 있습니다.</p>
<h3 id="거부된-대안-rejected-alternatives"><a href="#거부된-대안-rejected-alternatives">거부된 대안 (Rejected alternatives)</a></h3>
<h4 id="noop가-무언가를-반환하도록-하는-경우"><a href="#noop가-무언가를-반환하도록-하는-경우"><code>noop()</code>가 무언가를 반환하도록 하는 경우</a></h4>
<p>이 제안은 "YAGNI (You Ain't Gonna Need It, 필요 없을 것이다)" 원칙에 따라 거부되었습니다.</p>
<p>이는 의미를 복잡하게 만들기 때문입니다. 예를 들어, <code>*args</code>와 <code> **kws</code>를 항상 모두 반환한다면, 아무것도 주어지지 않았을 때 무엇을 반환해야 할까요? <code>((), {})</code> 형태의 튜플을 반환하는 것은 다소 보기 좋지 않지만 일관성을 제공합니다. 하지만 함수에 전달된 것이 없다는 개념적인 의미를 고려하면 단순히 <code>None</code>을 반환하는 것이 더 나을 수도 있습니다.</p>
<p>또는 <code>noop(7)</code>와 같이 하나의 위치 인자만 전달한다면 7을 반환해야 할까요, 아니면 <code>((7,), {})</code>를 반환해야 할까요? 이와 같은 문제들이 발생합니다.</p>
<p>작성자는 <code>noop()</code>의 반환 값이 필요하지 않을 것이므로 항상 <code>None</code>을 반환해야 한다고 주장합니다.</p>
<p>** Coghlan의 논의 (서식 편집):**</p>
<p>이에 대한 반론은 <code>map(noop, iterable)</code>, <code>sorted(iterable, key=noop)</code> 등 ( <code>filter</code>, <code>max</code>, <code>min</code>은 모두 단일 인자를 받는 호출 가능 객체를 허용하며, 많은 <code>itertools</code> 연산도 마찬가지입니다)에서 <code>noop()</code>를 유용한 기본 함수로 만들기 위해서는 다음과 같이 정의해야 한다는 것입니다.</p>
<pre><code class="language-python">def noop(*args, **kwds):
    return args[0] if args else None
</code></pre>
<p>이 반론에 대한 재반론은 이러한 모든 경우에 기본값으로 <code>None</code>을 사용하는 것이 더 빠르다는 것입니다. 왜냐하면 이렇게 하면 알고리즘이 콜백을 호출하여 아무것도 유용한 일을 하지 않게 하는 대신, 콜백을 완전히 건너뛸 수 있기 때문입니다.</p>
