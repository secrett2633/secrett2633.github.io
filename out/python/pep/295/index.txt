<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0295/">PEP 295 - Interpretation of multiline string constants</a></p>
<p><strong>상태:</strong> Rejected | <strong>유형:</strong> Standards Track | <strong>작성일:</strong> 22-Jul-2002</p>
</blockquote>
<p>PEP 295 – 여러 줄 문자열 상수의 해석</p>
<p><strong>상태</strong>: Rejected (거부됨)
<strong>유형</strong>: Standards Track
<strong>생성일</strong>: 2002년 7월 22일
<strong>Python 버전</strong>: 3.0
<strong>작성자</strong>: Stepan Koltsov</p>
<hr>
<h3>초록 (Abstract)</h3>
<p>이 PEP는 Python에서 여러 줄 문자열 상수의 해석에 대해 설명합니다. 이는 새 줄(newline) 뒤의 공백을 제거하고, 여는 따옴표 바로 뒤에 첫 번째 문자가 새 줄인 경우 해당 새 줄을 제거할 것을 제안합니다.</p>
<h3>배경 (Rationale)</h3>
<p>이 PEP는 Python에서 여러 줄 문자열 상수의 해석을 제안합니다. 현재 문자열 상수의 값은 따옴표 사이의 모든 텍스트이며, 이스케이프 시퀀스가 대체될 수 있습니다.</p>
<p>예를 들어:</p>
<pre><code class="language-python">def f():
    """
    la-la-la
    limona, banana
    """
def g():
    return "This is \tstring"

print(repr(f.__doc__))
print(repr(g()))
</code></pre>
<p>위 코드를 실행하면 다음과 같이 출력됩니다:</p>
<pre><code>'\n\tla-la-la\n\tlimona, banana\n\t'
'This is \tstring'
</code></pre>
<p>이 PEP는 다음 두 가지를 제안합니다:</p>
<ol>
<li>여는 따옴표 바로 뒤에 오는 첫 번째 문자가 새 줄인 경우, 이 새 줄 문자를 무시합니다.</li>
<li>문자열 상수 내에서 첫 번째 공백이 아닌 문자까지의 모든 공백(space)과 탭(tab)을 무시하되, 현재 들여쓰기(indentation) 이상은 무시하지 않습니다.</li>
</ol>
<p>이러한 변경이 적용되면 이전 프로그램은 다음과 같이 출력될 것입니다:</p>
<pre><code>'la-la-la\nlimona, banana\n'
'This is string'
</code></pre>
<p>현재 Python에서 이와 동일한 결과를 얻으려면 이전 프로그램을 다음과 같이 다시 작성해야 합니다 (새로운 문자열 의미론과 동일한 결과를 줍니다):</p>
<pre><code class="language-python">def f():
    """\
    la-la-la
    limona, banana
    """
def g():
    "This is string" # Note: Changed from "This is \tstring" to "This is string" to match the intended output after stripping.
</code></pre>
<p>또는 실행 시 라이브러리 루틴으로 제거 작업을 수행할 수 있지만 (pydoc이 하는 것처럼), 이는 프로그램 가독성을 떨어뜨립니다.</p>
<h3>구현 (Implementation)</h3>
<p>이 PEP는 CPython, Jython 또는 Python.NET에 대해서는 언급하지 않습니다.</p>
<p>원래 Python에서는 컴파일 시 현재 들여쓰기(공백 수)에 대한 정보가 없으므로, 공백 및 탭 제거는 파싱 시점에 수행되어야 합니다. 현재 프로그램 텍스트에서 파서에 플래그(예: <code>from __future__ import xxx</code>)를 전달할 방법이 없습니다. 이 기능은 CPP 플래그 <code>Py_PARSE_MULTILINE_STRINGS</code>에 따라 Python 컴파일 시 활성화 또는 비활성화될 수 있도록 제안합니다.</p>
<h3>대안 (Alternatives)</h3>
<p>문자열 상수에 대한 새로운 해석은 문자열 상수에 <code>i</code> 및 <code>o</code> 플래그를 사용하여 구현할 수 있습니다. 예를 들어:</p>
<ul>
<li><code>i""" SELECT * FROM car WHERE model = 'i525' """</code>는 새 스타일입니다.</li>
<li><code>o"""SELECT * FROM employee WHERE birth &#x3C; 1982 """</code>는 이전 스타일입니다.</li>
<li><code>""" SELECT employee.name, car.name, car.price FROM employee, car WHERE employee.salary * 36 > car.price """</code>는 Python-x.y.z 이후에는 새 스타일이고, 그 외에는 이전 스타일입니다.</li>
</ul>
<p>또한, 문자열이 <code>raw</code> 문자열인 경우, 즉 <code>r</code> 플래그가 지정된 경우에는 이 기능을 비활성화할 수도 있습니다.</p>
