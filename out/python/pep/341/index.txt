<h1>PEP 341 – <code>try-except</code>와 <code>try-finally</code>의 통합</h1>
<ul>
<li><strong>작성자:</strong> Georg Brandl</li>
<li><strong>상태:</strong> Final (최종)</li>
<li><strong>유형:</strong> Standards Track</li>
<li><strong>생성일:</strong> 2005년 5월 4일</li>
<li><strong>Python 버전:</strong> 2.5</li>
<li><strong>최종 수정일:</strong> 2025-02-01 08:59:27 GMT</li>
</ul>
<h2>초록 (Abstract)</h2>
<p>이 PEP는 <code>try</code> 문의 문법과 의미론에 대한 변경 사항을 제안하여, <code>try-except-finally</code> 블록을 결합할 수 있도록 합니다. 요약하자면, 다음과 같이 코드를 작성하는 것이 유효하게 됩니다.</p>
<pre><code class="language-python">try:
    &#x3C;do something>
except Exception:
    &#x3C;handle the error>
finally:
    &#x3C;cleanup>
</code></pre>
<h2>배경 및 제안 (Rationale/Proposal)</h2>
<p><code>try-except</code> 문과 <code>try-finally</code> 문은 각각 많은 사용 사례를 가지고 있습니다. 그러나 종종 예외를 포착(catch)하고 그 후에 일부 정리(cleanup) 코드를 실행해야 하는 경우가 있습니다. 이 경우, 현재는 다음과 같이 작성해야 하므로 다소 번거롭고 이해하기 어렵습니다.</p>
<pre><code class="language-python">f = None
try:
    try:
        f = open(filename)
        text = f.read()
    except IOError:
        print 'An error occurred'
    finally:
        if f:
            f.close()
</code></pre>
<p>따라서, 다음과 같은 구조가</p>
<pre><code class="language-python">try:
    &#x3C;suite 1>
except Ex1:
    &#x3C;suite 2>
&#x3C;more except: clauses>
else:
    &#x3C;suite 3>
finally:
    &#x3C;suite 4>
</code></pre>
<p>레거시(legacy) 방식인 다음 구조와 정확히 동일하게 작동하도록 제안됩니다.</p>
<pre><code class="language-python">try:
    try:
        &#x3C;suite 1>
    except Ex1:
        &#x3C;suite 2>
    &#x3C;more except: clauses>
    else:
        &#x3C;suite 3>
finally:
    &#x3C;suite 4>
</code></pre>
<p>이 변경은 하위 호환성을 유지하며, 현재 유효한 모든 <code>try</code> 문은 계속해서 작동합니다.</p>
<h2>문법 변경 (Changes to the grammar)</h2>
<p><code>try</code> 문의 문법은 현재 다음과 같습니다.</p>
<pre><code>try_stmt: ('try' ':' suite (except_clause ':' suite)+ ['else' ':' suite] | 'try' ':' suite 'finally' ':' suite)
</code></pre>
<p>이것은 다음과 같이 변경되어야 합니다.</p>
<pre><code>try_stmt: 'try' ':' suite ( (except_clause ':' suite)+ ['else' ':' suite] ['finally' ':' suite] | 'finally' ':' suite )
</code></pre>
<h2>구현 (Implementation)</h2>
<p>이 PEP의 작성자는 현재 CPython 구현에 대한 충분한 지식이 없었기 때문에, 직접 구현을 제공할 수 없었습니다. Thomas Lee가 패치를 제출했습니다.</p>
<p>하지만 Guido에 따르면, 이는 구현하기 매우 쉬운 작업이라고 합니다. – 적어도 핵심 개발자(core hacker)에게는 그렇습니다.</p>
