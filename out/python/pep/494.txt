<h2 id="pep-494-python-36-릴리스-일정"><a href="#pep-494-python-36-릴리스-일정">PEP 494: Python 3.6 릴리스 일정</a></h2>
<p>이 문서는 Python 3.6의 개발 및 릴리스 일정을 설명하는 Python Enhancement Proposal (PEP)입니다. 이 PEP는 주로 PEP 단위의 항목들에 중점을 둡니다.</p>
<h3 id="개요-abstract"><a href="#개요-abstract">개요 (Abstract)</a></h3>
<p>PEP 494는 Python 3.6 버전의 개발 및 릴리스 일정을 상세히 기술합니다.</p>
<h3 id="릴리스-관리자-및-담당자-release-manager-and-crew"><a href="#릴리스-관리자-및-담당자-release-manager-and-crew">릴리스 관리자 및 담당자 (Release Manager and Crew)</a></h3>
<p>Python 3.6 릴리스를 담당한 주요 인력은 다음과 같습니다.</p>
<ul>
<li><strong>3.6 릴리스 관리자 (Release Manager):</strong> Ned Deily</li>
<li><strong>Windows 인스톨러 (installers):</strong> Steve Dower</li>
<li><strong>Mac 인스톨러 (installers):</strong> Ned Deily</li>
<li><strong>문서화 (Documentation):</strong> Julien Palard, Georg Brandl</li>
</ul>
<h3 id="36-수명-주기-36-lifespan"><a href="#36-수명-주기-36-lifespan">3.6 수명 주기 (3.6 Lifespan)</a></h3>
<p>Python 3.6은 약 24개월 동안 약 3개월마다 버그 수정(bugfix) 업데이트를 받았습니다. 3.7.0 final 버전이 릴리스된 후, 3.6의 최종 버그 수정 업데이트가 제공되었습니다. 그 이후에는 3.6 final 릴리스 후 5년까지, 즉 대략 2021년 12월까지 필요에 따라 보안 업데이트(security updates, 소스 코드만 해당)가 릴리스될 것으로 예상되었습니다.</p>
<p>2021년 12월 23일을 기점으로, 3.6 버전은 릴리스 수명 주기의 "End-of-Life (EOL)" 단계에 도달했습니다. 3.6.15가 마지막 보안 릴리스였으며, 3.6 코드 베이스는 현재 동결(frozen)되어 더 이상의 업데이트는 제공되지 않으며 버그 트래커(bug tracker)에서도 어떠한 종류의 이슈도 더 이상 접수되지 않습니다.</p>
<h3 id="릴리스-일정-release-schedule"><a href="#릴리스-일정-release-schedule">릴리스 일정 (Release Schedule)</a></h3>
<p>다음은 Python 3.6의 주요 릴리스 일정입니다.</p>

<h3 id="36의-주요-기능-features-for-36"><a href="#36의-주요-기능-features-for-36">3.6의 주요 기능 (Features for 3.6)</a></h3>
<p>3.6.0 beta 1 시점에 구현된 주요 변경 사항 및 기능들은 다음과 같습니다. 이 기능들은 Python 3.6의 개발 방향과 개선점을 이해하는 데 중요합니다.</p>
<ul>
<li><strong>PEP 468, <code>Keyword Argument</code> 순서 보존 (Preserving Keyword Argument Order):</strong> 함수 호출 시 전달된 키워드 인자들의 순서를 유지합니다.</li>
<li><strong>PEP 487, 클래스 생성 사용자 정의 간소화 (Simpler customization of class creation):</strong> 클래스 생성(metaclass)을 더욱 쉽게 사용자 정의할 수 있도록 합니다.</li>
<li><strong>PEP 495, 지역 시간 모호성 해소 (Local Time Disambiguation):</strong> 일광 절약 시간제(DST) 전환 시 발생하는 시간 모호성을 처리합니다.</li>
<li><strong>PEP 498, 리터럴 문자열 포맷팅 (Literal String Formatting):</strong> F-string (formatted string literals) 도입으로 문자열 포맷팅이 더욱 간결해졌습니다.</li>
<li><strong>PEP 506, 표준 라이브러리에 <code>Secrets</code> 모듈 추가 (Adding A Secrets Module To The Standard Library):</strong> 암호학적으로 강력한 난수 생성을 위한 <code>secrets</code> 모듈이 추가되었습니다.</li>
<li><strong>PEP 509, <code>dict</code>에 비공개 버전 추가 (Add a private version to dict):</strong> <code>dict</code> 객체 내부 구현에 대한 최적화 및 안정성 개선.</li>
<li><strong>PEP 515, 숫자 리터럴에 밑줄 (Underscores in Numeric Literals):</strong> 숫자 리터럴 중간에 밑줄(<code>_</code>)을 사용하여 가독성을 높일 수 있습니다 (예: <code>1_000_000</code>).</li>
<li><strong>PEP 519, 파일 시스템 경로 프로토콜 추가 (Adding a file system path protocol):</strong> <code>os.PathLike</code> 프로토콜을 도입하여 경로 객체를 더욱 유연하게 다룰 수 있게 합니다.</li>
<li><strong>PEP 520, 클래스 속성 정의 순서 보존 (Preserving Class Attribute Definition Order):</strong> 클래스 내에 정의된 속성들의 순서가 유지됩니다.</li>
<li><strong>PEP 523, CPython에 프레임 평가 API 추가 (Adding a frame evaluation API to CPython):</strong> CPython 내부의 프레임(frame) 평가를 위한 새로운 API를 제공합니다.</li>
<li><strong>PEP 524, Linux에서 <code>os.urandom()</code>을 블로킹 방식으로 변경 (시작 중) (Make os.urandom() blocking on Linux (during system startup)):</strong> 시스템 시작 중 <code>os.urandom()</code>이 충분한 엔트로피(entropy)를 얻을 때까지 블로킹하여 보안을 강화합니다.</li>
<li><strong>PEP 525, 비동기 제너레이터 (Asynchronous Generators) (잠정적):</strong> <code>async def</code> 문법을 사용하여 비동기 제너레이터를 정의할 수 있게 합니다.</li>
<li><strong>PEP 526, 변수 어노테이션 구문 (Syntax for Variable Annotations) (잠정적):</strong> 변수에 타입 어노테이션(type annotation)을 지정하는 새로운 구문이 도입되었습니다.</li>
<li><strong>PEP 528, Windows 콘솔 인코딩을 UTF-8로 변경 (Change Windows console encoding to UTF-8) (잠정적):</strong> Windows 콘솔의 기본 인코딩을 UTF-8로 변경하여 유니코드(Unicode) 지원을 개선합니다.</li>
<li><strong>PEP 529, Windows 파일 시스템 인코딩을 UTF-8로 변경 (Change Windows filesystem encoding to UTF-8) (잠정적):</strong> Windows 파일 시스템의 인코딩을 UTF-8로 변경하여 국제화(internationalization)를 강화합니다.</li>
<li><strong>PEP 530, 비동기 컴프리헨션 (Asynchronous Comprehensions):</strong> <code>async for</code>를 <code>List Comprehension</code>, <code>Set Comprehension</code>, <code>Dictionary Comprehension</code>, <code>Generator Expression</code>에서 사용할 수 있도록 합니다.</li>
</ul>
