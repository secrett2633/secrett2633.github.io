<h2>PEP 452 – 암호화 해시 함수용 API v2.0 번역 및 정리</h2>
<h3>개요 (Abstract)</h3>
<p>MD5 또는 SHA와 같은 암호화 해싱 알고리즘을 구현하는 여러 모듈이 존재합니다. 이 PEP는 이러한 알고리즘들을 위한 표준 API를 지정하여, 여러 구현체 간에 쉽게 전환할 수 있도록 하는 것을 목표로 합니다.</p>
<h3>명세 (Specification)</h3>
<p>모든 해싱 모듈은 동일한 인터페이스를 제공해야 합니다. 추가적인 메서드나 변수를 더할 수 있지만, 이 문서에 설명된 요소들은 항상 존재해야 합니다.</p>
<h4>해시 함수 모듈 정의:</h4>
<ol>
<li>
<p><strong><code>new([string])</code> (키가 없는 해시)</strong>
<strong><code>new(key, [string], [digestmod])</code> (키가 있는 해시)</strong>
새로운 해싱 객체를 생성하여 반환합니다.</p>
<ul>
<li>첫 번째 형식은 MD5나 SHA와 같이 키가 없는 해시에 사용됩니다.</li>
<li>HMAC과 같이 키가 있는 해시의 경우, <code>key</code>는 사용할 키를 담은 <code>bytes-like object</code>를 포함하는 필수 매개변수입니다.</li>
<li>두 경우 모두, 선택적 <code>string</code> 매개변수가 제공되면, <code>obj.update(string)</code>이 호출된 것처럼 객체의 초기 상태로 즉시 해싱됩니다.</li>
<li>해싱 객체를 생성한 후에는 <code>update()</code> 메서드를 사용하여 임의의 바이트를 객체에 공급할 수 있으며, <code>digest()</code> 메서드를 호출하여 언제든지 해시 값을 얻을 수 있습니다.</li>
<li>매개변수 이름은 <code>string</code>이지만, 해싱 객체는 8비트 데이터만 처리합니다. <code>key</code>와 <code>string</code> 모두 <code>bytes-like object</code> (예: <code>bytes</code>, <code>bytearray</code>)여야 합니다. Python 3.x에서는 텍스트(유니코드)는 더 이상 지원되지 않습니다.</li>
<li>이 함수에 임의의 추가 키워드 인수를 추가할 수 있지만, 제공되지 않을 경우 합리적인 기본값이 사용되어야 합니다.</li>
</ul>
</li>
<li>
<p><strong><code>digest_size</code> (모듈 변수)</strong>
정수 값으로, 이 모듈이 생성하는 해싱 객체가 생성하는 다이제스트(digest)의 크기를 바이트 단위로 나타냅니다. 가변 출력 크기를 가진 해시는 이 변수를 <code>None</code>으로 설정합니다.</p>
</li>
</ol>
<h4>해싱 객체 요구 사항:</h4>
<ol>
<li>
<p><strong><code>digest_size</code> (객체 속성)</strong>
모듈 수준의 <code>digest_size</code> 변수와 동일하게, 해싱 객체가 생성하는 다이제스트의 크기를 바이트 단위로 측정합니다. 해시가 가변 출력 크기를 갖는 경우, 이 출력 크기는 해싱 객체가 생성될 때 선택되어야 하며, 이 속성에는 선택된 크기가 포함되어야 합니다. 따라서 이 속성에 <code>None</code>은 허용되지 않습니다.</p>
</li>
<li>
<p><strong><code>block_size</code></strong>
정수 값 또는 <code>NotImplemented</code>입니다. 해시 알고리즘의 내부 블록 크기를 바이트 단위로 나타냅니다. <code>HMAC</code> 모듈에서 비밀 키를 <code>digest_size</code>로 채우거나 <code>digest_size</code>보다 길면 비밀 키를 해싱하는 데 사용됩니다. 해시 알고리즘에 대한 <code>HMAC</code> 알고리즘이 표준화되지 않은 경우 <code>NotImplemented</code>를 반환합니다.</p>
</li>
<li>
<p><strong><code>name</code></strong>
텍스트 문자열 값으로, 해싱 알고리즘의 표준화된 소문자 이름입니다. 이 이름은 <code>hashlib.new</code>에 적합한 매개변수여야 합니다.</p>
</li>
</ol>
<h4>해싱 객체 메서드 요구 사항:</h4>
<ol>
<li>
<p><strong><code>copy()</code></strong>
이 해싱 객체의 독립적인 복사본을 반환합니다. 이 복사본에 대한 업데이트는 원래 객체에 영향을 미치지 않습니다.</p>
</li>
<li>
<p><strong><code>digest()</code></strong>
이 해싱 객체의 해시 값을 8비트 데이터를 포함하는 <code>bytes</code>로 반환합니다. 이 함수에 의해 객체는 어떤 식으로든 변경되지 않으며, 이 함수를 호출한 후에도 객체를 계속 업데이트할 수 있습니다.</p>
</li>
<li>
<p><strong><code>hexdigest()</code></strong>
이 해싱 객체의 해시 값을 16진수 숫자를 포함하는 문자열로 반환합니다. <code>a</code>부터 <code>f</code>까지의 숫자는 소문자를 사용해야 합니다. <code>.digest()</code> 메서드와 마찬가지로, 이 메서드는 객체를 변경해서는 안 됩니다.</p>
</li>
<li>
<p><strong><code>update(string)</code></strong>
<code>bytes-like object</code>인 <code>string</code>을 해싱 객체의 현재 상태로 해싱합니다. <code>update()</code>는 해싱 객체의 수명 동안 여러 번 호출될 수 있습니다.</p>
</li>
</ol>
<h3>배경 (Rationale)</h3>
<ul>
<li><strong><code>digest_size</code>는 바이트 단위:</strong> 해시 알고리즘 크기는 보통 비트 단위로 인용되지만(<code>MD5</code>는 128비트 알고리즘), <code>digest_size</code>는 바이트 단위로 측정됩니다. 이는 파일에서 탐색하거나 출력 문자열의 길이를 계산하는 등 바이트 길이가 더 자주 필요하기 때문입니다. 비트 단위 크기가 필요한 소수의 사람들은 <code>digest_size</code>에 8을 곱해야 합니다.</li>
<li><strong><code>update()</code> 메서드 이름 유지:</strong> <code>update()</code> 메서드 이름을 <code>append()</code>로 변경하자는 제안이 있었지만, 이 메서드는 실제 해싱 객체의 현재 상태를 업데이트하는 역할을 하며, Python에 포함된 <code>md5</code> 및 <code>sha</code> 모듈에서 이미 <code>update()</code>를 사용하고 있으므로 이름을 변경하지 않는 것이 가장 간단하다고 판단되었습니다.</li>
<li><strong>키가 있는 해시의 생성자 인수 순서:</strong> 키가 있는 해시의 생성자 인수는 <code>key</code>가 필수 매개변수이므로 일반적으로 필수가 먼저 오도록 <code>new(key, [string])</code> 순서가 결정되었습니다. 이는 <code>string</code> 매개변수가 첫 번째 위치에서 두 번째로 이동하는 것을 의미합니다.</li>
</ul>
<h3>버전 1.0에서 2.0으로의 변경 사항 (Changes from Version 1.0 to Version 2.0)</h3>
<p>API for Cryptographic Hash Functions의 버전 2.0은 API의 일부 측면을 명확히 하고 최신 상태로 만들었습니다. 또한 사실상 표준으로 이미 대부분의 구현체에서 제공되던 측면을 공식화했습니다.</p>
<p>버전 2.0은 다음의 새로운 속성을 도입했습니다.</p>
<ul>
<li><strong><code>name</code></strong> : 이슈 18532에 의해 <code>name</code> 속성이 필수가 되었습니다.</li>
<li><strong><code>block_size</code></strong> : 새로운 버전은 반환 값 <code>NotImplemented</code>가 <code>HMAC</code> 지원을 방지하도록 지정합니다.</li>
</ul>
<p>버전 2.0은 Python 3.0에서 이진 데이터와 텍스트 데이터의 분리를 고려했습니다. <code>new()</code> 및 <code>update()</code>의 <code>string</code> 인수와 <code>key</code> 인수는 <code>bytes-like objects</code>여야 합니다. Python 2.x에서는 해싱 객체가 ASCII 전용 유니코드를 지원할 수도 있습니다. 인수의 실제 이름은 공용 API의 일부이므로 변경되지 않았습니다.</p>
