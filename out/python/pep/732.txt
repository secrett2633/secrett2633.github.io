<p><strong>원문 링크:</strong> <a href="https://peps.python.org/pep-0732/" target="_blank" rel="noopener noreferrer">PEP 732 - The Python Documentation Editorial Board</a></p>

<h2 id="pep-732-python-문서-편집-위원회"><a href="#pep-732-python-문서-편집-위원회">PEP 732 – Python 문서 편집 위원회</a></h2>
<h3 id="개요-abstract"><a href="#개요-abstract">개요 (Abstract)</a></h3>
<p>이 PEP(Python Enhancement Proposal)는 Python 문서 편집 위원회(Python Documentation Editorial Board)를 설립하고, 이 편집 위원회가 어떻게 운영될 것인지 제안합니다.</p>
<h3 id="배경-motivation"><a href="#배경-motivation">배경 (Motivation)</a></h3>
<p>2021년 3월, Steering Council은 문서의 방향을 설정하기 위해 Documentation Working Group (문서 워킹 그룹)의 생성을 승인했습니다. 이 그룹은 이후 보다 전술적인 작업에 중점을 두기 위해 생성된 다른 Documentation Working Group과 구별하기 위해 'Editorial Board'로 명칭이 변경되었습니다.</p>
<p>Python 문서의 목적은 현재와 미래의 Python 최종 사용자에게 봉사하는 것입니다. 이를 위해 핵심 개발 커뮤니티와 더 넓은 Python 문서 기여자들이 함께 협력합니다.</p>
<h3 id="상세-내용-specification"><a href="#상세-내용-specification">상세 내용 (Specification)</a></h3>
<h4 id="임무-mandate"><a href="#임무-mandate">임무 (Mandate)</a></h4>
<p>편집 위원회는 다음을 수행합니다.</p>
<ul>
<li>Python 문서의 품질을 유지하고 개선하기 위한 프로세스가 마련되도록 보장합니다.</li>
<li>현재와 미래의 사용자에게 봉사하는 커뮤니티 리소스(community resource)로서 Python 문서가 발전하도록 지원합니다.</li>
<li>Python 프로그래밍 언어를 발전시키고, 다양하고 국제적인 Python 프로그래머 커뮤니티의 성장을 지원하고 촉진하는 Python Software Foundation (PSF)의 미션(mission)에 따라 행동합니다.</li>
<li>문서 기여가 접근 가능하고(accessible), 포괄적이며(inclusive), 지속 가능하도록(sustainable) 보장합니다.</li>
<li>문서 콘텐츠에 대한 적절한 의사 결정 프로세스를 확립합니다.</li>
<li>결정을 내리기 전에 기여자들 간의 합의를 이루려고 노력합니다.</li>
<li>문서 콘텐츠 결정에 대한 최종 중재자(final arbiter) 역할을 합니다.</li>
</ul>
<h4 id="책임-responsibilities"><a href="#책임-responsibilities">책임 (Responsibilities)</a></h4>
<p>위원회는 아래에 명시된 범위 내에서 Python 문서에 대한 결정을 내릴 권한을 가집니다. 예를 들어, 다음을 수행할 수 있습니다.</p>
<ul>
<li>Python 문서의 큰 그림 전략을 설정합니다.</li>
<li>문서의 의도된 구조를 설정합니다.</li>
<li>글쓰기와 디자인 모두에 대한 스타일 및 편집 결정을 내립니다.</li>
<li>문서 거버넌스(governance)를 처리합니다 (예: 의사 결정을 주제 전문가에게 위임, 의견 불일치 해결, 결정 등).</li>
</ul>
<h4 id="범위-scope"><a href="#범위-scope">범위 (Scope)</a></h4>
<p>편집 위원회는 다음 내용에 대한 콘텐츠 및 전략을 감독합니다.</p>
<ul>
<li><strong>포함 범위 (In scope):</strong>
<ul>
<li>CPython 문서 (<code>docs.python.org</code>)</li>
<li>CPython 개발 가이드 (<code>devguide.python.org</code>)</li>
<li>CPython 문서 번역</li>
<li>PEPs (<code>peps.python.org</code>)</li>
</ul>
</li>
<li><strong>제외 범위 (Not in scope):</strong>
<ul>
<li>CPython 코드베이스 내 코드 주석 (Code comments in CPython codebase)</li>
<li>CPython docstrings (주석 문자열)</li>
<li>PyPA 문서</li>
<li><code>www.python.org</code></li>
<li>Python Wiki (<code>wiki.python.org</code>)</li>
</ul>
</li>
</ul>
<h4 id="구성-composition"><a href="#구성-composition">구성 (Composition)</a></h4>
<p>Python 문서 편집 위원회는 5명의 위원으로 구성됩니다.</p>
<h5 id="편집-위원회-위원-editorial-board-members"><a href="#편집-위원회-위원-editorial-board-members">편집 위원회 위원 (Editorial Board Members)</a></h5>
<p>초기 편집 위원회 위원은 다음과 같습니다.</p>
<ul>
<li>Mariatta Wijaya</li>
<li>Ned Batchelder</li>
<li>Joanna Jablonski</li>
<li>Guido van Rossum</li>
<li>Carol Willing</li>
</ul>
<h5 id="편집-위원회-위원-자격-editorial-board-member-qualifications"><a href="#편집-위원회-위원-자격-editorial-board-member-qualifications">편집 위원회 위원 자격 (Editorial Board Member Qualifications)</a></h5>
<p>편집 위원회 위원은 다음을 갖춰야 합니다.</p>
<ul>
<li>Python 프로젝트 철학에 대한 깊은 이해</li>
<li>Python 교육 및 개발자 대상 문서화 경험</li>
<li>건설적이고 도움이 되는 뛰어난 기여 기록</li>
<li>Python에 대한 상당한 기여 이력</li>
<li>Python 문서 개선에 시간을 할애하려는 의지</li>
</ul>
<p>편집 위원회 위원은 교육, 커뮤니케이션, 기술 문서 작성, Python 문서, 접근성(accessibility), 번역 또는 커뮤니티 관리 경험이 있어야 합니다.</p>
<h5 id="임기-term"><a href="#임기-term">임기 (Term)</a></h5>
<p>편집 위원회 위원의 임기는 정해져 있지 않지만, 일반적으로 매년 편집 위원회 구성에 변화가 있을 것으로 예상됩니다. 위원들은 매년 위원직을 계속할 의사가 있는지 확인합니다. 위원들은 언제든지 사임할 수 있습니다.</p>
<p>만약 위원이 한 달 이상 연락이 닿지 않거나 연락이 되지 않는 경우, 나머지 위원들이 투표를 통해 교체를 결정할 수 있습니다.</p>
<h5 id="편집-위원회-규모-변경-changes-to-the-editorial-boards-size"><a href="#편집-위원회-규모-변경-changes-to-the-editorial-boards-size">편집 위원회 규모 변경 (Changes to the Editorial Board's Size)</a></h5>
<p>매년 주요 Python 릴리스(release) 이후, 편집 위원회는 위원회 규모를 변경해야 하는지 검토합니다. 이는 문서 커뮤니티의 필요가 시간이 지남에 따라 변할 경우 유연성을 제공합니다. 위원회 규모를 늘리기로 결정하려면 현재 위원회의 80%가 참여하는 정족수(quorum) 하에 단순 과반수가 필요합니다.</p>
<p>문서 편집 위원회의 후원 조직으로서 Steering Council은 언제든지 위원회 위원의 수를 변경할 수 있으며, 여기에는 새로운 위원 임명 또는 기존 위원 해고가 포함됩니다.</p>
<h5 id="공석-vacancies"><a href="#공석-vacancies">공석 (Vacancies)</a></h5>
<p>어떤 이유로든 위원회에 공석이 발생하면, 문서 편집 위원회는 잠재적인 위원 후보를 공개적으로 모집합니다. 후보자들은 자격과 봉사 동기를 간략히 기술한 문서를 제출해야 합니다. 현재 편집 위원회 위원들은 현재 위원회의 80%가 참여하는 정족수 하에 단순 과반수 투표로 새로운 위원을 선출합니다.</p>
<h4 id="개정-amendments"><a href="#개정-amendments">개정 (Amendments)</a></h4>
<p>이 PEP는 문서 편집 위원회의 헌장 역할을 합니다. 위원회 운영에 대한 변경 사항은 새로운 PEP를 통하거나 이 PEP를 변경함으로써 이루어질 수 있습니다. 두 경우 모두, 변경 사항은 커뮤니티 논의 후 Steering Council에 의해 결정됩니다.</p>
<h4 id="pep-수락-pep-acceptance"><a href="#pep-수락-pep-acceptance">PEP 수락 (PEP Acceptance)</a></h4>
<p>PEP 732는 2023년 12월 11일 Python Steering Council에 의해 수락되었습니다.</p>
<p>Steering Council은 PEP에 명시된 범위에 동의하지만, 편집 위원회가 충분히 확립되고 더 높은 우선순위가 처리된 후에는 표준 라이브러리의 docstrings를 포함하도록 범위를 확장하는 것을 고려하는 것이 합리적일 것이라고 언급했습니다.</p>
<h4 id="문의-contact"><a href="#문의-contact">문의 (Contact)</a></h4>
<p>편집 위원회에 결정을 요청하려면, 커뮤니티 구성원은 <code>python/editorial-board</code> 저장소에 이슈를 열 수 있습니다.</p>
