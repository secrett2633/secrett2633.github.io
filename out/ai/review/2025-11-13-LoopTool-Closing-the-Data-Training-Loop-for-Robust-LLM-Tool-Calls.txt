<p><strong>저자:</strong> Kangning Zhang, Wenxiang Jiao, Kounianhua Du, Yuan Lu, Weiwen Liu, Weinan Zhang, Yong Yu</p>
<h2 id="핵심-연구-목표"><a href="#핵심-연구-목표">핵심 연구 목표</a></h2>
<p>기존 LLM 툴 학습의 <strong>정적 합성 데이터 파이프라인</strong> 이 모델의 약점에 적응하지 못하고 노이즈 있는 레이블을 유지하여 훈련 효율성을 저해하는 문제를 해결합니다. 고비용의 클로즈드-소스 API 의존성을 제거하고, 데이터 생성과 모델 훈련을 긴밀하게 통합하는 <strong>완전 자동화된 모델-인지 데이터 진화 프레임워크</strong> 를 구축하여 LLM의 툴 사용 능력을 강화하는 것을 목표로 합니다.</p>
<h2 id="핵심-방법론"><a href="#핵심-방법론">핵심 방법론</a></h2>
<p><strong>LoopTool</strong> 은 데이터-훈련 루프를 닫는 반복적인 프레임워크로, <strong>GRPO 기반 사후 훈련</strong> 과 세 가지 모듈을 통합합니다. 첫째, <strong>Greedy Capability Probing (GCP)</strong> 은 모델의 숙달 및 실패한 역량을 진단하여 학습 가치가 높은 샘플을 식별합니다. 둘째, <strong>Judgement-Guided Label Verification (JGLV)</strong> 은 <strong>오픈소스 판정 모델(Qwen3-32B)</strong> 을 활용하여 어노테이션 오류를 찾아 수정하고 데이터셋을 정화합니다. 셋째, <strong>Error-Driven Data Expansion (EDDE)</strong> 은 식별된 실패 사례를 기반으로 구조적으로 유사하면서도 맥락적으로 다양한 새로운 도전적 샘플을 생성하며, 이 모든 과정은 <strong>오픈소스 모델</strong> 을 통해 저비용으로 이루어집니다.</p>
<h2 id="주요-결과"><a href="#주요-결과">주요 결과</a></h2>
<p><strong>LoopTool로 훈련된 8B 모델</strong> 은 <strong>32B 데이터 생성기</strong> 의 성능을 크게 뛰어넘었으며, BFCL-v3 벤치마크에서 <strong>74.93%의 전체 정확도</strong> 를 달성하여 해당 규모에서 <strong>최고 수준의 오픈소스 모델 중 3위</strong> 를 기록했습니다. ACEBench 벤치마크에서는 <strong>73.4%의 전체 정확도</strong> 로 <strong>8B 규모 오픈소스 모델 중 최고의 성능</strong> 을 보였습니다. 또한 <strong>Qwen3-8B</strong> 대비 BFCL-v3에서 <strong>+8.59%p</strong> , ACEBench에서 <strong>+6.3%p</strong> 향상을 보여주며, 비-툴 관련 도메인에서도 <strong>일반화 능력</strong> 을 유지하거나 개선했습니다.</p>
<h2 id="ai-실무자를-위한-시사점"><a href="#ai-실무자를-위한-시사점">AI 실무자를 위한 시사점</a></h2>
<p><strong>폐쇄 루프, 자가 개선 데이터 파이프라인</strong> 이 LLM의 툴 사용 능력을 극적으로 향상시킬 수 있음을 입증했습니다. <strong>고비용의 상용 API 의존성 없이 오픈소스 모델</strong> 만으로 데이터 생성 및 평가를 수행할 수 있음을 보여주어, 비용 효율적인 AI 개발 방향을 제시합니다. <strong>오류 주도 데이터 확장</strong> 및 <strong>레이블 검증</strong> 을 통해 모델의 약점을 동적으로 보완하고 데이터 품질을 높이는 실용적인 전략을 제공하여, 더욱 견고하고 다재다능한 AI 에이전트 구축에 기여합니다.</p>
