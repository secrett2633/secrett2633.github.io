<p><strong>링크:</strong> <a href="https://arxiv.org/abs/2602.10999">논문 PDF로 바로 열기</a></p>
</blockquote>
<p><strong>저자:</strong> Yusong Lin, Haiyang Wang, Shuzhe Wu, Lue Fan, Feiyang Pan, Sanyuan Zhao, Dandan Tu</p>
<h2>핵심 연구 목표</h2>
<p>본 논문은 실세계 소프트웨어 개발에 필수적인 <strong>CLI(명령줄 인터페이스) 환경과의 상호작용</strong> 을 포함하는 <strong>환경 집약적 에이전트 작업</strong> 의 확장 가능한 데이터 생성 파이프라인 부재 문제를 해결하고자 합니다. 기존 <strong>SWE-bench</strong> 와 같은 코드 중심 작업과 달리, <strong>Terminal-Bench</strong> 와 같은 환경 집약적 작업은 수동 생성에 의존하여 규모가 작고 LLM 에이전트의 성능이 40% 미만으로 저조한 한계가 있었습니다.</p>
<h2>핵심 방법론</h2>
<p>저자들은 <strong>CLI-Gym</strong> 이라는 파이프라인을 제안하며, 에이전트가 <strong>"골드" 환경</strong> (GitHub 리포지토리와 Dockerfile로 정의된 건강한 상태)에서 시작하여 의도적으로 오류를 유발하는 <strong>환경 인버스 과정</strong> 을 시뮬레이션합니다. <strong>LLM 기반 에이전트</strong> 는 실행 피드백의 안내를 받아 환경을 조작하는 <strong>Dockerfile 명령</strong> 을 생성하고, <strong>유닛 테스트 실패</strong> 가 감지되면 오류 유발 Dockerfile과 실패한 유닛 테스트 정보, LLM이 생성한 문제 설명을 결합하여 새로운 작업을 만듭니다. 최종적으로 <strong>291개의 고품질 궤적</strong> 을 선별한 후, <strong>Qwen3-32B</strong> 및 <strong>Qwen3-235B-A22B-Instruct 모델</strong> 을 사전 훈련된 <strong>SWE-스타일 궤적</strong> 과 CLI-Gym 궤적에 파인튜닝했습니다.</p>
<h2>주요 결과</h2>
<p><strong>CLI-Gym</strong> 은 <strong>1,655개</strong> 의 환경 집약적 CLI 작업 인스턴스를 생성했으며, 이는 기존 수동 작업의 약 20배 규모입니다. 파인튜닝된 <strong>LiberCoder-32B</strong> 모델은 <strong>Terminal-Bench 1.0</strong> 에서 <strong>38.9% Pass@1</strong> 을 달성하며 기준 모델 대비 <strong>+28.6%</strong> 향상을 보였고, <strong>LiberCoder-235B-A22B</strong> 는 <strong>46.1% Pass@1</strong> 을 달성하여 <strong>+21.1%</strong> 향상을 기록했습니다. 이 모델들은 <strong>Kimi-K2 (1조 파라미터)</strong> 및 <strong>Qwen3-Coder-480B (480B 파라미터)</strong> 와 같은 더 큰 오픈소스 모델들을 능가하는 성능을 보였습니다.</p>
<h2>AI 실무자를 위한 시사점</h2>
<p><strong>CLI-Gym</strong> 은 AI 에이전트의 <strong>환경 집약적 작업 수행 능력</strong> 을 향상시키기 위한 <strong>확장 가능한 데이터 생성 솔루션</strong> 을 제공합니다. 이는 LLM 에이전트가 소프트웨어 개발에서 실제 시스템 문제를 진단하고 해결하는 데 필수적인 역량을 강화하는 데 기여합니다. 특히, <strong>고품질의 소규모 도메인 특화 데이터</strong> (예: 291개 궤적)로도 <strong>LLM의 성능을 크게 향상</strong> 시킬 수 있음을 입증하여, 데이터 양뿐만 아니라 <strong>데이터 품질과 다양성</strong> 의 중요성을 강조합니다.</p>
