<p><strong>저자:</strong> Mingde Xu, Zhen Yang, Wenyi Hong, Lihang Pan, Xinyue Fan, Yan Wang, Xiaotao Gu, Bin Xu, Jie Tang</p>
<h2>핵심 연구 목표</h2>
<p>본 논문은 기존 <strong>Vision-Language Models (VLMs)</strong> 기반의 UI-to-Code 접근 방식이 정적인 HTML/CSS 코드만 생성하고 GUI 상호작용을 지원하지 못하는 한계를 극복하고자 합니다. 사용자와의 상호작용이 가능한 실행 가능한 UI 코드를 자동으로 생성하고 검증하는 <strong>에이전트 기반 프레임워크</strong> 를 구축하여 실제 UI 개발 워크플로우에 통합될 수 있도록 하는 것이 목표입니다.</p>
<h2>핵심 방법론</h2>
<p>제안하는 <strong>WebVIA 프레임워크</strong> 는 세 가지 핵심 구성요소로 이루어져 있습니다. 첫째, <strong>탐색 에이전트(WebVIA-Agent)</strong> 는 HTML 환경과 상호작용하여 다중 상태 UI 스크린샷과 상호작용 그래프를 수집합니다. 둘째, <strong>UI2Code 모델(WebVIA-UI2Code)</strong> 은 수집된 스크린샷을 활용하여 실행 가능한 대화형 HTML/CSS/JavaScript 코드를 생성합니다. 셋째, <strong>검증 모듈</strong> 은 생성된 GUI의 상호작용성과 기능을 평가합니다. 에이전트와 UI2Code 모델은 대규모 합성 GUI 상호작용 데이터셋을 기반으로 <strong>감독 학습(SFT)</strong> 을 통해 훈련됩니다.</p>
<h2>주요 결과</h2>
<p><strong>WebVIA-Agent</strong> 는 <strong>UIExplore-Bench</strong> 에서 <strong>Gemini-2.5-Pro</strong> 와 같은 범용 에이전트보다 더 안정적이고 정확한 UI 탐색 성능을 보이며, 종합 점수 <strong>89.8%</strong> , <strong>완전성(Completeness) 93.1%</strong> , <strong>정확성(Correctness) 97.7%</strong> 를 달성했습니다. 또한, <strong>WebVIA-UI2Code-Qwen</strong> 및 <strong>WebVIA-UI2Code-GLM</strong> 모델은 <strong>UIFlow2Code-Bench</strong> 에서 각각 <strong>75.9%</strong> 와 <strong>84.9%</strong> 의 성능을 기록하며, 베이스라인 모델을 크게 능가하여 실행 가능하고 상호작용적인 코드를 성공적으로 생성했습니다.</p>
<h2>AI 실무자를 위한 시사점</h2>
<p><strong>WebVIA 프레임워크</strong> 는 AI/ML 엔지니어에게 정적 UI를 넘어 <strong>동적이고 상호작용 가능한 웹 인터페이스</strong> 를 자동 생성하는 새로운 가능성을 제시합니다. 특히, <strong>에이전트 기반 접근 방식</strong> 과 <strong>상호작용 검증 모듈</strong> 은 UI 개발의 효율성을 높이고 생성된 코드의 신뢰성을 확보하는 데 기여할 수 있습니다. 다만, 현재는 <strong>Click, Enter, Select</strong> 와 같은 제한된 액션 유형과 주로 합성 데이터에 의존한 학습은 실제 복잡한 웹 환경에 대한 일반화 능력을 높이기 위한 추가적인 연구가 필요함을 시사합니다.</p>
