<p><strong>저자:</strong> Yifan Zhang and Lanser Contributors</p>
<h2>핵심 연구 목표</h2>
<p>대규모 언어 모델(LLM) 기반의 언어 에이전트가 코드 관련 태스크에서 겪는 API 환각 및 코드 변경 오류 문제를 해결하고자 합니다. 이를 위해 언어 서버(Language Server)가 제공하는 검증된 코드 정보와 <strong>기계 검증 가능한 단계별 프로세스 보상(process reward)</strong> 을 에이전트에 제공하여, 에이전트의 계획 루프를 실제 프로그램 현실과 일치시키는 것을 목표로 합니다.</p>
<h2>핵심 방법론</h2>
<p><code>Lanser-CLI</code>라는 CLI-first 오케스트레이션 레이어를 제안합니다. 이는 LSP(Language Server Protocol) 서버와 연동하여 (i) 코드 편집에 강건한 <strong>Selector DSL</strong> (심볼릭, AST-경로, 콘텐츠 앵커) 기반의 <strong>결정론적 재배치 알고리즘</strong> 을 통해 정확한 코드 요소 주소를 지정합니다. (ii) 서버 응답을 표준화하고 환경 메타데이터를 포함하는 <strong>결정론적 Analysis Bundles</strong> 를 생성하여 재현 가능한 아티팩트를 보장합니다. (iii) 미리 보기, 워크스페이스 격리, Git 인식 트랜잭션 적용 등의 <strong>다층 안전 장치</strong> 를 통해 코드 변경 작업의 안전성을 확보합니다. (iv) <strong>진단 감소(diagnostic reduction)</strong> , <strong>안전성 통과 여부(safety checks)</strong> , <strong>해소된 모호성(disambiguation confidence)</strong> 을 기반으로 한 <strong>프로세스 보상 함수</strong> (<code>rt = α(Dt-1 - Dt) + β St - γ (1 - at)</code>)를 정의하여 에이전트의 중간 단계를 감독합니다.</p>
<h2>주요 결과</h2>
<p><code>Lanser-CLI</code>는 언어 서버 상호작용을 <strong>검증 가능하고 재현 가능한 아티팩트(Analysis Bundles)</strong> 로 변환하여, 모든 실행에서 동일한 <code>bundleId</code>를 생성하는 <strong>결정론성</strong> 을 입증했습니다. 특히, 제안된 프로세스 보상 함수는 <strong>진단 감소 시 긍정적인 보상(예: 1.894)</strong> 을, <strong>모호성 또는 안전성 문제 시 부정적인 보상(예: -0.438)</strong> 을 제공함을 예시를 통해 보여주었습니다. 이는 에이전트의 단계별 의사결정에 대한 <strong>실용적인 피드백 메커니즘</strong> 을 제공하며, <strong>불변성 조건 하에 보상의 단조성</strong> 을 이론적으로 증명했습니다.</p>
<h2>AI 실무자를 위한 시사점</h2>
<p>이 연구는 AI 에이전트가 코드를 더욱 <strong>정확하고 안전하며 신뢰할 수 있게</strong> 다룰 수 있는 방법을 제시합니다. <code>Lanser-CLI</code>의 <strong>결정론적 처리</strong> 와 <strong>강력한 안전 장치</strong> 는 LLM 기반 코드 에이전트의 배포에 필요한 신뢰성을 크게 향상시킬 수 있습니다. 특히, <strong>프로세스 보상</strong> 은 강화 학습 에이전트가 복잡한 코드 수정 작업을 수행할 때 <strong>효율적인 학습 신호</strong> 로 활용되어, LLM이 단순히 텍스트 예측을 넘어 실제 프로그래밍 환경에 깊이 관여하도록 돕는 중요한 발전입니다.</p>
