<h2 id="핵심-연구-목표"><a href="#핵심-연구-목표">핵심 연구 목표</a></h2>
<p>기존 모바일 GUI 에이전트 벤치마크인 <strong>AndroidWorld</strong> 의 포화 상태(90% 이상의 성공률)와 현실적이지 않은 태스크 복잡성 한계를 극복하는 것을 목표로 합니다. 특히, 모호한 사용자 지침과 외부 도구(MCP) 사용 시나리오를 반영하지 못하는 문제를 해결하고, 실제 모바일 사용을 더 잘 반영하는 도전적이고 재현 가능한 벤치마크를 제시하고자 합니다.</p>
<h2 id="핵심-방법론"><a href="#핵심-방법론">핵심 방법론</a></h2>
<p>새로운 벤치마크 <strong>MobileWorld</strong> 는 20개 애플리케이션에 걸쳐 201개의 태스크로 구성되며, <strong>평균 27.8단계</strong> 의 긴 실행 경로와 <strong>62.2%</strong> 의 다중 애플리케이션 태스크를 포함합니다. <strong>에이전트-사용자 상호작용(Agent-User Interaction)</strong> 태스크와 <strong>Model Context Protocol (MCP)-augmented tasks</strong> 를 도입하여 각각 모호한 지침 처리 및 GUI와 외부 도구(총 <strong>61개</strong> ) 통합 능력을 평가합니다. <strong>Docker-in-Docker 컨테이너</strong> 환경과 <strong>스냅샷 기반 상태 관리</strong> , 백엔드 데이터베이스 및 애플리케이션 콜백을 통한 <strong>결정론적 검증</strong> 을 특징으로 합니다.</p>
<h2 id="주요-결과"><a href="#주요-결과">주요 결과</a></h2>
<p><strong>MobileWorld</strong> 는 기존 벤치마크 대비 현저히 높은 난이도를 보였으며, 최신 <strong>Agentic Framework (GPT-5 + UI-Ins-7B)</strong> 의 <strong>전반적 성공률은 51.7%</strong> 로, AndroidWorld에서의 90% 이상과 비교할 때 큰 성능 격차를 보였습니다. 특히, <strong>에이전트-사용자 상호작용 태스크</strong> 에서 <strong>GPT-5</strong> 가 <strong>62.2%</strong> 의 성공률을, <strong>MCP-augmented 태스크</strong> 에서는 <strong>51.6%</strong> 를 달성했으나, Doubao-1.5-UI-TARS와 같은 end-to-end 모델은 사용자 상호작용 및 MCP 호출 기능 지원 부족으로 해당 카테고리에서 저조한 성능을 기록했습니다.</p>
<h2 id="ai-실무자를-위한-시사점"><a href="#ai-실무자를-위한-시사점">AI 실무자를 위한 시사점</a></h2>
<p><strong>MobileWorld</strong> 는 현세대 모바일 에이전트가 <strong>사용자 모호성 감지 및 해명, MCP 컨텍스트 관리, 장기 기억 및 상태 추적, 복잡한 논리 추론, 시공간적 상황 인식</strong> 과 같은 핵심 연구 격차를 가지고 있음을 명확히 보여줍니다. <strong>LLM 기반의 Agentic Framework</strong> 가 end-to-end 모델보다 우수한 성능을 보이며, 복잡한 모바일 자동화에서 <strong>능동적인 사용자 상호작용</strong> 과 <strong>외부 도구 통합</strong> 의 중요성을 강조합니다. 따라서 향후 연구는 <strong>기반 모델 개선</strong> 과 <strong>컨텍스트 길이 제한 해결</strong> , <strong>MCP 도구 관리 혁신</strong> 에 집중해야 할 것입니다.</p>
