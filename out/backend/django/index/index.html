[Django] index

2025년 2월 14일수정: 2025년 2월 14일

Django에서 데이터베이스 인덱스를 효율적으로 사용하는 방법에 대해 알아보겠습니다.

인덱스란?

인덱스는 데이터베이스에서 데이터를 빠르게 찾을 수 있도록 도와주는 자료구조입니다.

Django에서 인덱스 사용하기

1. 단일 필드 인덱스

from django.db import models

class Article(models.Model):
    title = models.CharField(max_length=100, db_index=True)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True, db_index=True)
    
    class Meta:
        db_table = 'article'

2. 복합 인덱스

class User(models.Model):
    name = models.CharField(max_length=50)
    email = models.EmailField()
    age = models.IntegerField()
    
    class Meta:
        db_table = 'user'
        indexes = [
            models.Index(fields=['name', 'email']),
            models.Index(fields=['age', 'name']),
        ]

3. 커스텀 인덱스

class Product(models.Model):
    name = models.CharField(max_length=100)
    category = models.CharField(max_length=50)
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        db_table = 'product'
        indexes = [
            models.Index(
                fields=['category', 'price'],
                name='product_category_price_idx'
            ),
            models.Index(
                fields=['-created_at'],  # 내림차순
                name='product_created_desc_idx'
            ),
        ]

인덱스 최적화 팁

1. 자주 사용되는 쿼리에 맞는 인덱스 생성

# 자주 사용되는 쿼리
Article.objects.filter(category='django', status='published')

# 해당 쿼리에 맞는 인덱스
class Meta:
    indexes = [
        models.Index(fields=['category', 'status']),
    ]

2. 부분 인덱스 사용

class Meta:
    indexes = [
        models.Index(
            fields=['status'],
            condition=models.Q(status='published'),
            name='published_articles_idx'
        ),
    ]

3. 함수 기반 인덱스

from django.contrib.postgres.indexes import GinIndex
from django.contrib.postgres.search import SearchVector

class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    
    class Meta:
        indexes = [
            GinIndex(
                SearchVector('title', 'content'),
                name='article_search_idx'
            ),
        ]

인덱스 모니터링

1. 사용되지 않는 인덱스 확인

-- PostgreSQL에서 사용되지 않는 인덱스 확인
SELECT 
    schemaname,
    tablename,
    indexname,
    idx_scan,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes
WHERE idx_scan = 0;

2. 인덱스 크기 확인

-- 인덱스 크기 확인
SELECT 
    indexname,
    pg_size_pretty(pg_relation_size(indexname::regclass)) as size
FROM pg_indexes
WHERE tablename = 'article';

주의사항

  1. 인덱스는 저장 공간을 차지합니다
  2. INSERT, UPDATE, DELETE 성능에 영향을 줍니다
  3. 너무 많은 인덱스는 오히려 성능을 저하시킬 수 있습니다

적절한 인덱스 사용으로 Django 애플리케이션의 성능을 크게 향상시킬 수 있습니다!
