9:Ta7e,<h2><strong>들어가며</strong></h2>
<p>Django에서 데이터베이스에 접근할 때, 우리는 주로 ORM(Object-Relational Mapping)을 사용합니다.<br>
ORM을 사용하면 쿼리문을 직접 작성하지 않고도 데이터베이스와 상호작용할 수 있어 매우 편리하지만,<br>
쿼리 로그를 확인하기 어려워 최적화가 필요한 부분을 파악하기 힘들 수 있습니다.</p>
<p>이번 포스트에서는 Django에서 쿼리 로그를 확인하는 방법에 대해 소개하려고 합니다.<br>
쿼리 로그를 활용하면 ORM이 어떻게 동작하는지 파악하고 성능 최적화에 도움을 받을 수 있습니다.</p>
<hr>
<h2><strong>쿼리 로그 확인하기</strong></h2>
<h3><strong>1. 환경 설정</strong></h3>
<p>먼저, <code>settings.py</code> 파일에 다음과 같이 설정을 추가합니다. 이 설정은 Django에서 쿼리 로그를 콘솔에 출력할 수 있도록 도와줍니다.</p>
<pre><code class="language-python">LOGGING = {
    "version": 1,
    "disable_existing_loggers": False,
    "handlers": {
        "console": {
            "class": "logging.StreamHandler",  # 콘솔에 로그를 출력하는 핸들러
        },
    },
    "loggers": {
        "django.db.backends": {  # 데이터베이스 쿼리 로그를 출력하는 로거
            "handlers": ["console"],  # 콘솔 핸들러로 로그를 출력
            "level": "DEBUG",  # 로그 레벨을 DEBUG로 설정하여 자세한 쿼리 정보 출력
        },
    },
}
</code></pre>
<p>이 설정을 추가하면 Django는 데이터베이스 쿼리 로그를 콘솔에 출력하게 됩니다.<br>
이제 개발 중에 ORM이 실행하는 쿼리를 직접 확인할 수 있게 됩니다.</p>
<h3><strong>2. 쿼리 로그 확인하기</strong></h3>
<p>설정을 마친 후, 실제로 Django에서 쿼리 로그를 확인할 수 있습니다.<br>
개발 중에 ORM을 통해 쿼리를 실행하면, 그 쿼리가 콘솔에 출력됩니다.</p>
<p>예를 들어, <code>save</code>를 사용할 때 실행되는 쿼리들을 확인해보았습니다.<br>
이 방법을 통해 쿼리 로그를 보면 ORM이 어떻게 동작하는지 쉽게 파악할 수 있습니다.</p>
